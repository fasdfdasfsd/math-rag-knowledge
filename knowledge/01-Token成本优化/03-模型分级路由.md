# 模型分级路由

## 概述

根据任务复杂度将请求路由到不同级别的模型，在保证质量的前提下最小化成本。采用三级路由策略：Haiku 级（deepseek-v4-flash 等轻量模型）处理简单任务，Sonnet 级处理中等复杂度任务，Opus 级（deepseek-v4-pro 等旗舰模型）处理最复杂的架构设计和推理任务。

## 核心规则

### 🔴 必须遵守 (MUST)

1. **所有任务必须按复杂度分级路由**
   - 不允许所有任务都使用最贵的旗舰模型
   - 每个请求必须有明确的模型选择依据

2. **子代理模型默认使用 flash 级别**
   - 子代理（sub-agent）处理的是分解后的子任务，复杂度通常较低
   - 除非子代理需要执行复杂推理，否则不得升级模型

3. **必须维护模型路由决策树的可观测性**
   - 记录每次请求的模型选择、任务类型、实际 token 消耗
   - 定期审计模型路由决策的合理性

### 🟡 强烈建议 (SHOULD)

1. **任务分级决策树**
   - **Haiku 级 (flash)**：代码补全、简单问答、格式化、语法检查、正则表达式生成
   - **Sonnet 级**：中等复杂度代码生成、文档编写、代码审查、bug 修复
   - **Opus 级 (pro)**：架构设计、系统设计、复杂算法、安全审计、多文件重构

2. **成本监控与告警**
   - 设置月度成本预算
   - 单次请求 token 消耗超过阈值时告警
   - 如果 flash 级别足够完成任务但路由到 pro，视为违规

### 🟢 可选建议 (MAY)

1. **自动降级策略**
   - flash 回答质量不足时自动升级到 sonnet
   - sonnet 仍不足时再升级到 pro

2. **A/B 测试**
   - 对模糊任务同时请求 flash 和 sonnet 对比结果

## 正确示例

```python
from enum import Enum
from dataclasses import dataclass
import time


class ModelTier(str, Enum):
    HAUKA = "deepseek-v4-flash"       # 低成本，适合简单任务
    SONNET = "claude-sonnet-4-20250514"  # 平衡质量与成本
    OPUS = "deepseek-v4-pro"           # 高质量，适合复杂任务


class TaskComplexity(str, Enum):
    SIMPLE = "simple"         # 代码补全、格式化、简单问答
    MODERATE = "moderate"     # bug修复、文档编写、中等代码
    COMPLEX = "complex"       # 架构设计、系统设计、安全审计


@dataclass
class TaskRequest:
    task_type: str
    complexity: TaskComplexity
    estimated_tokens: int
    requires_reasoning: bool = False
    is_sub_agent: bool = False


class ModelRouter:
    """模型分级路由器"""
    
    def __init__(self):
        self.usage_log: list[dict] = []
    
    def select_model(self, request: TaskRequest) -> ModelTier:
        """根据任务复杂度选择模型"""
        
        # 子代理默认用 Haiku 级
        if request.is_sub_agent and not request.requires_reasoning:
            return ModelTier.HAUKA
        
        decision_tree = {
            TaskComplexity.SIMPLE: ModelTier.HAUKA,
            TaskComplexity.MODERATE: ModelTier.SONNET,
            TaskComplexity.COMPLEX: ModelTier.OPUS,
        }
        
        model = decision_tree.get(request.complexity, ModelTier.SONNET)
        
        # 记录路由决策
        self._log_routing(request, model)
        return model
    
    def _log_routing(self, request: TaskRequest, model: ModelTier):
        self.usage_log.append({
            "timestamp": time.strftime("%Y-%m-%dT%H:%M:%SZ", time.gmtime()),
            "task_type": request.task_type,
            "complexity": request.complexity.value,
            "selected_model": model.value,
            "estimated_tokens": request.estimated_tokens,
        })
    
    def get_cost_report(self) -> dict:
        """生成成本报告"""
        total_cost = 0
        tier_counts = {tier: 0 for tier in ModelTier}
        
        for entry in self.usage_log:
            tier = ModelTier(entry["selected_model"])
            tier_counts[tier] += 1
            total_cost += self._estimate_cost(
                tier, entry["estimated_tokens"]
            )
        
        return {
            "total_requests": len(self.usage_log),
            "tier_distribution": tier_counts,
            "estimated_cost_usd": round(total_cost, 4),
        }
    
    @staticmethod
    def _estimate_cost(tier: ModelTier, tokens: int) -> float:
        """估算成本（仅供参考，实际价格以官方为准）"""
        costs = {
            ModelTier.HAUKA: 1.0 / 1_000_000,    # $1/M tokens
            ModelTier.SONNET: 3.0 / 1_000_000,   # $3/M tokens
            ModelTier.OPUS: 15.0 / 1_000_000,    # $15/M tokens
        }
        return tokens * costs[tier]


# 使用示例
router = ModelRouter()

tasks = [
    TaskRequest("代码补全", TaskComplexity.SIMPLE, 500),
    TaskRequest("Bug修复", TaskComplexity.MODERATE, 2000),
    TaskRequest("架构设计", TaskComplexity.COMPLEX, 5000),
    TaskRequest("文件搜索", TaskComplexity.SIMPLE, 200, is_sub_agent=True),
]

for task in tasks:
    model = router.select_model(task)
    print(f"[{task.task_type}] -> {model.value}")
    
print("\n成本报告:", router.get_cost_report())
```

## 错误示例

```python
# 错误 1：所有任务都用最贵模型
def bad_router_all_opus():
    """每次请求都用 Opus/Pro"""
    task_types = [
        "查询数据库字段名",
        "格式化 JSON",
        "写一行注释",
        "生成 UUID",
    ]
    for task in task_types:
        # 这些简单任务 flash 完全能胜任
        print(f"[{task}] -> deepseek-v4-pro")  # 浪费！
    # 成本差异：flash ~$0.001 vs pro ~$0.015 (10-15x)


# 错误 2：子代理使用旗舰模型
def bad_sub_agent_routing():
    """子代理做简单任务也用 pro"""
    agent_configs = [
        {"name": "search_agent", "model": "deepseek-v4-pro"},  # 错误
        {"name": "format_agent", "model": "deepseek-v4-pro"},  # 错误
    ]
    # 子代理任务通常是独立的、范围有限的任务
    # 默认应使用 flash，除非有明确理由


# 错误 3：无路由逻辑，硬编码模型
def bad_hardcoded_model():
    """代码中硬编码模型"""
    import anthropic
    client = anthropic.Anthropic()
    
    # 所有功能都用同一个模型
    model_name = "claude-sonnet-4-20250514"  # 硬编码
    
    # 实际上应根据功能区分
    # - 代码搜索 → flash
    # - 代码生成 → sonnet
    # - 架构审查 → pro


# 错误 4：没有成本监控
def bad_no_monitoring():
    """
    没有记录任何成本指标
    月末才知道花了多少钱
    无法按功能模块分摊成本
    无法识别异常的高消耗请求
    """
```

## 工具链配置

```python
# 自动降级策略 — 在质量不达标时升级
class AutoEscalationRouter(ModelRouter):
    """带自动升级的路由器"""
    
    MIN_CONFIDENCE_THRESHOLD = 0.7  # 答案质量评分阈值
    
    def route_with_escalation(self, request: TaskRequest) -> tuple[ModelTier, str]:
        """启动模型 → 检查质量 → 必要时升级"""
        current_tier = self.select_model(request)
        result = self._call_model(current_tier, request)
        
        quality_score = self._assess_quality(result)
        if quality_score < self.MIN_CONFIDENCE_THRESHOLD:
            # 自动升级到下一个级别
            upgraded_tier = self._upgrade(current_tier)
            if upgraded_tier != current_tier:
                result = self._call_model(upgraded_tier, request)
                return upgraded_tier, result
        
        return current_tier, result
```

## 进阶专题

### 2026 年价格与性能矩阵

| 模型 | 输入/Mt | 输出/Mt | 适用 | 备注 |
|------|:--:|:--:|------|------|
| **Sonnet 4.6** | $3 | $15 | **默认 — 90% 任务** | 性价比之王 |
| **Opus 4.6** | $5 | $25 | 复杂架构、深度调试 | 仅真难题 |
| **Opus Fast Mode** | $30 | $150 | **避免** | 6× 溢价，"奢侈税" |
| **Haiku 4.5** | $1 | $5 | 简单任务、子代理 | 轻量级 |

### 关键配置调优

| 配置 | 效果 |
|------|------|
| `effortLevel: medium` | 推理 token 减少 **20-30%**，日常任务质量无损 |
| `CLAUDE_AUTOCOMPACT_PCT_OVERRIDE=80` | 提前触发压缩，避开"最差状态做总结" |
| `CLAUDE_CODE_MAX_CONTEXT_TOKENS=100000` | 限制窗口上限，强制早压缩 |
| `CLAUDE_CODE_DISABLE_FAST_MODE=1` | 禁用 6× 奢侈税 |

### 子代理委托模式

```
主会话 (Sonnet)  →  子代理 A (Haiku) — 代码搜索、格式化
                   →  子代理 B (Sonnet) — Bug 修复、Code Review
                   →  子代理 C (Opus)  — 架构审查、安全审计

主会话只收摘要，中间过程不污染上下文
```

**黄金法则**：子代理默认 Haiku，只在任务需要时升级。

### 缓存与模型切换的关系

**永不中途切换模型**。缓存按模型隔离——从 Opus 切 Sonnet 意味着整个缓存重建，可能比继续用 Opus 更贵。需要不同模型时，用子代理。
