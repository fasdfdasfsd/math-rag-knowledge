# 通信协议选型规范

## 一、协议对比

| 维度 | REST | gRPC | GraphQL | WebSocket | MCP |
|------|------|------|---------|-----------|-----|
| 协议 | HTTP/1.1 | HTTP/2 | HTTP/1.1 | WS over HTTP | HTTP |
| 数据格式 | JSON | Protobuf | JSON | 任意 | JSON-RPC |
| 类型安全 | OpenAPI | 内置 | Schema | 无 | JSON Schema |
| 流式传输 | ❌ | ✅ 双向 | ❌ | ✅ 双向 | ❌ |
| 浏览器支持 | ✅ | ⚠️ 需代理 | ✅ | ✅ | ❌ |
| 缓存 | ✅ HTTP缓存 | ❌ | ❌ | ❌ | ❌ |
| 学习曲线 | 低 | 中 | 中 | 低 | 低 |
| 工具生态 | 丰富 | 丰富 | 中等 | 中等 | 早期 |

## 二、选型决策树

```
需要浏览器直接调用？
├── 是 → REST + OpenAPI
└── 否 → 需要高性能服务间通信？
    ├── 是 → gRPC + Protobuf
    └── 否 → 客户端需要灵活查询？
        ├── 是 → GraphQL
        └── 否 → AI Agent 调用？
            ├── 是 → MCP (Model Context Protocol)
            └── 否 → REST
```

## 三、各协议使用规范

### REST
- **适用**：公开/合作伙伴 API、浏览器直接调用
- **契约**：OpenAPI 3.1+ spec，Spectral linting
- **版本**：URL 路径版本（`/v1/`）
- **错误**：RFC 9457 Problem Details
- **禁止**：GET 请求修改状态、URL 中包含动词

### gRPC
- **适用**：高吞吐内部服务间通信
- **契约**：Protobuf `.proto` 文件
- **版本**：包级别（`user.v1`, `user.v2`）
- **规则**：不修改已有字段号，删除字段用 `reserved`
- **禁止**：在同一个 `.proto` 文件混用不同版本的消息

### GraphQL
- **适用**：移动端/前端灵活数据需求
- **安全**：查询深度限制、复杂度分析
- **性能**：DataLoader 避免 N+1
- **禁止**：无限制的递归查询、暴露内部数据结构

### WebSocket
- **适用**：实时推送（消息通知、协作编辑）
- **认证**：连接时验证 token，超时断开
- **重连**：exponential backoff + jitter
- **禁止**：无心跳检测的长连接、单连接发送过多数据

### MCP (Model Context Protocol)
- **适用**：AI Agent 调用外部工具和数据源
- **安全**：工具权限最小化、输入校验
- **禁止**：暴露敏感数据源、无速率限制

## 四、版本与兼容

- 所有协议变更评估：源兼容、线上兼容、语义兼容
- 破坏性变更 → 主版本号 + 迁移指南
- 至少保留 6 个月废弃通知
