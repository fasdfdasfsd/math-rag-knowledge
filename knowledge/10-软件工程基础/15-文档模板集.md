# 文档模板集

## 一、ADR 模板（MADR 4.0 格式）

```yaml
---
status: proposed | accepted | deprecated | superseded
date: YYYY-MM-DD
deciders: [决策者]
consulted: [咨询者]
informed: [知会者]
---
```

```markdown
# ADR-XXXX: <简短标题>

## 背景与问题陈述
<当前状态、触发决策的事件>

## 决策驱动因素
- <驱动因素 1>
- <驱动因素 2>

## 考虑的方案
| 方案 | 优点 | 缺点 |
|------|------|------|
| 方案 A | ... | ... |
| 方案 B | ... | ... |

## 决策结果
**选择方案 X**，因为 <核心理由>。

## 后果
### 正面
- <正面影响>
### 负面
- <需要接受的权衡>
### 中性
- <中立影响>

## 确认方式
<如何验证决策被正确实施>
```

**规则**：
- 已接受的 ADR 不可修改，只能被新的 ADR 取代
- 存放位置：`docs/adr/`，与代码同仓库

## 二、PR 模板

```markdown
## 变更摘要
<!-- 3-6 条，简洁描述 -->

## 动机
<!-- 解决什么问题？ -->

## 范围边界
<!-- 明确不做什么 -->

## 风险等级：[低 | 中 | 高]
<!-- 判断依据 -->

## 测试证据
<!-- 如何验证？未测试什么？ -->

## 上线方案
<!-- 特性开关？灰度？迁移步骤？ -->

## 截图/日志（如适用）

## 后续事项
<!-- 已知债务或后续 PR -->
```

## 三、代码审查检查清单

### 通用审查
- [ ] 变更范围与 PR 描述一致
- [ ] 错误路径已处理
- [ ] 无硬编码密钥/配置
- [ ] 日志级别合理

### 高风险变更附加检查
适用于：认证/支付/数据迁移/基础设施/安全

- [ ] 授权逻辑已验证（Anti-BOLA）
- [ ] 数据迁移有回滚方案
- [ ] 基础设施变更已 terraform plan
- [ ] 安全扫描通过（SAST + 依赖扫描）

## 四、Release Notes / Changelog 模板

```markdown
# Changelog
遵循 [Keep a Changelog](https://keepachangelog.com/) 和 [SemVer](https://semver.org/)。

## [1.2.0] - YYYY-MM-DD

### Added
- 新功能 X

### Changed
- 依赖 A 从 v2 升级到 v3

### Deprecated
- 端点 B（将在 2.0.0 移除）

### Removed
- 功能 C（已在 1.1.0 废弃）

### Fixed
- Bug D (#1234)

### Security
- 修补漏洞 E (CVE-XXXX-XXXXX)
```

**规则**：
- Changelog 是给人看的，不是给机器看的
- 最新版本在最上面
- 日期用 ISO 8601：`YYYY-MM-DD`
- 版本标题链接到 diff URL

```markdown
<!-- 威胁建模摘要 -->

## 测试策略

## 上线计划

## 替代方案
<!-- 考虑过但未选择的方案 -->
```
