# 数据结构与算法规范

## 概述

本规范提供常用数据结构的复杂度参考、Python 内置数据结构的最佳使用方式、算法选择指南以及应避免的不良实践。目标是指导开发者在日常编码中做出高效的数据结构选择，避免低效实现。

---

## 核心要点

### 常用数据结构复杂度速查表

#### 时间复杂度

| 数据结构 | 访问 | 搜索 | 插入 | 删除 | 说明 |
|----------|------|------|------|------|------|
| Array（数组） | O(1) | O(n) | O(n) | O(n) | 连续内存，随机访问快 |
| Stack（栈） | O(n) | O(n) | O(1) | O(1) | 后进先出（LIFO）|
| Queue（队列） | O(n) | O(n) | O(1) | O(1) | 先进先出（FIFO）|
| LinkedList（链表） | O(n) | O(n) | O(1) | O(1) | 已知节点指针时插入快 |
| Hash Table（哈希表） | O(1)* | O(1)* | O(1)* | O(1)* | *均摊平均情况 |
| Binary Search Tree | O(log n) | O(log n) | O(log n) | O(log n) | 平衡情况下 |
| B-Tree | O(log n) | O(log n) | O(log n) | O(log n) | 数据库索引常用 |
| Heap（堆） | O(1) | O(n) | O(log n) | O(log n) | 最小/最大值访问 O(1) |

#### 空间复杂度

| 数据结构 | 空间复杂度 | 额外开销 |
|----------|-----------|----------|
| Array | O(n) | 无指针开销 |
| LinkedList | O(n) | 每个节点额外存储 1-2 个指针 |
| Hash Table | O(n) | 装载因子通常 0.7，需额外空间 |
| Tree | O(n) | 每个节点 2-3 个指针 |

### Python 内置数据结构使用指南

| 类型 | 类名 | 可变 | 有序 | 唯一 | 适用场景 |
|------|------|------|------|------|----------|
| 列表 | `list` | 是 | 是 | 否 | 通用有序集合、栈、队列 |
| 元组 | `tuple` | 否 | 是 | 否 | 不可变序列、函数多返回值 |
| 字典 | `dict` | 是 | 是 (3.7+) | 键唯一 | 键值映射、缓存、计数器 |
| 集合 | `set` | 是 | 否 | 是 | 去重、集合运算 |
| 冻结集合 | `frozenset` | 否 | 否 | 是 | 作为字典键、不可变集合 |

**标准库扩展**：

| 模块 | 数据结构 | 用途 |
|------|----------|------|
| `collections.deque` | 双端队列 | O(1) 两端插入/删除 |
| `collections.Counter` | 计数器 | 元素频次统计 |
| `collections.defaultdict` | 默认字典 | 缺失键自动初始化 |
| `collections.OrderedDict` | 有序字典 | 保留插入顺序（Python 3.7+ dict 已内置此功能）|
| `heapq` | 堆 | 优先队列、Top-K |
| `bisect` | 二分搜索 | 有序列表插入与查找 |
| `array` | 数组 | 紧凑数值存储 |

### 算法选择决策树

```
数据量小（<1000）？           ──→ 直观算法即可（线性搜索、冒泡排序不推荐）
    否
需要排序？                   ──→ Timsort（Python 默认排序，自适应）
    否
需要搜索单个元素？           ──→ 已排序 → 二分搜索（bisect）
                               未排序 → 哈希集合辅助
    否
需要 Top-K 元素？            ──→ 堆（heapq.nlargest / nsmallest）
    否
大量字符串搜索？             ──→ Trie / Aho-Corasick
    否
图遍历？                     ──→ BFS（最短路径） / DFS（全遍历）
```

## 实际应用

### 禁止行为

| 禁止行为 | 说明 | 替代方案 |
|----------|------|----------|
| O(n²) 嵌套循环遍历大数据集 | 双重循环在 n > 10,000 时性能急剧下降 | 使用哈希表（dict/set）将 O(n²) 降为 O(n) |
| 不合理递归深度 | Python 默认递归深度限制 1000 | 改为迭代或手动设置 `sys.setrecursionlimit` |
| `for i in range(len(lst))` | 非 Pythonic 且易出错 | 直接 `for item in lst` 或 `enumerate` |
| 字符串拼接 `+=` 大量数据 | 字符串不可变，产生大量临时对象 | 使用 `str.join()` 或 `io.StringIO` |
| 用 `list` 实现队列 | `list.pop(0)` 是 O(n) 操作 | 使用 `collections.deque` |
| 用 `list` 实现去重 | 频繁 `if x not in list` 是 O(n) | 使用 `set` |

### Python 性能小贴士

- **局部变量访问**比全局变量快（局部变量在函数栈帧中，使用 LOAD_FAST）
- **列表推导式**比 `for` 循环 + `append` 快约 2 倍
- **dict 和 set** 底层是哈希表，查找 O(1)，多用它们替换 O(n) 线性搜索
- **生成器表达式**在处理大数据流时节省内存
- **`map`/`filter`** 在简单函数时快于列表推导式，但有 lambda 时更慢
