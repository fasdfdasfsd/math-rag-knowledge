# uv 包管理器使用

## 概述

uv 是 Astral 公司开发的现代 Python 包管理器，用 Rust 编写，比传统 pip 快 10-100 倍。uv 同时支持虚拟环境管理、依赖锁文件、项目初始化等功能，是 Python 项目的推荐包管理工具。

## 核心规则

### 🔴 必须遵守 (MUST)

1. **项目必须使用 uv 作为包管理器**
   - 新项目统一使用 uv，不再使用 pip
   - 禁止在同一项目中混用 pip 和 uv

2. **依赖声明在 pyproject.toml 中**
   - 使用 `uv add` 添加依赖，自动更新 pyproject.toml
   - 使用 `uv sync` 同步依赖，生成 uv.lock

3. **必须提交 uv.lock 到版本控制**
   - uv.lock 是确定性锁文件，确保环境可复现
   - 团队所有成员使用同一个 uv.lock

### 🟡 强烈建议 (SHOULD)

1. **虚拟环境管理**
   - 项目根目录下使用 `uv venv` 创建虚拟环境
   - 使用 `.venv` 作为默认虚拟环境目录

2. **依赖分组**
   - `dev` 分组：开发依赖（pytest, ruff, mypy）
   - `docs` 分组：文档构建依赖
   - 生产环境只安装核心依赖

3. **Python 版本管理**
   - 在 pyproject.toml 中声明 `requires-python`
   - 使用 `uv python pin` 固定 Python 版本

### 🟢 可选建议 (MAY)

1. **全局工具安装**
   - 使用 `uv tool install` 安装全局 CLI 工具
   - 替代 `pip install --user`

2. **项目脚手架**
   - 使用 `uv init` 快速创建新项目

## 正确示例

```bash
# ========== 项目初始化 ==========

# 初始化新项目
uv init my-project
cd my-project

# 创建虚拟环境
uv venv

# 激活虚拟环境（Windows PowerShell）
.venv\Scripts\Activate.ps1
# 或 (Unix bash)
# source .venv/bin/activate


# ========== 依赖管理 ==========

# 添加核心依赖
uv add fastapi uvicorn[standard] pydantic

# 添加开发依赖（dev 分组）
uv add --dev pytest pytest-asyncio ruff mypy

# 添加可选依赖（docs 分组）
uv add --group docs mkdocs mkdocs-material

# 同步依赖（安装所有依赖）
uv sync

# 只同步核心依赖（不含 dev/docs）
uv sync --no-dev


# ========== 版本控制 ==========

# 查看已安装的包
uv pip list

# 升级依赖
uv add fastapi@latest

# 删除依赖
uv remove some-package

# 查看依赖树
uv tree
```

生成的 `pyproject.toml`:

```toml
[project]
name = "my-project"
version = "0.1.0"
description = "Add your description here"
requires-python = ">=3.11"
dependencies = [
    "fastapi>=0.115.0",
    "pydantic>=2.0.0",
    "uvicorn[standard]>=0.30.0",
]

[project.optional-dependencies]
dev = [
    "pytest>=8.0",
    "pytest-asyncio>=0.24",
    "ruff>=0.6",
    "mypy>=1.11",
]
docs = [
    "mkdocs>=1.6",
    "mkdocs-material>=9.5",
]

[tool.uv]
dev-dependencies = ["rag-stack[dev]"]

[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"
```

## 错误示例

```bash
# 错误 1：混用 pip 和 uv

# 先用 uv 安装了依赖
uv add fastapi

# 然后又用 pip 安装
pip install requests  # 错误！

# uv 管理的虚拟环境和 pip 管理的环境可能不一致
# uv.lock 不会记录 pip 安装的依赖
# 导致其他开发者 uv sync 后缺少 requests


# 错误 2：不使用锁文件

# 项目中有 pyproject.toml，但没提交 uv.lock
# gitignore 中写了 uv.lock

# 后果：
# - 不同时间安装可能得到不同版本的依赖
# - CI 构建不可复现
# - 团队成员依赖版本不一致


# 错误 3：直接修改 pyproject.toml 而不使用 uv add

# 手动编辑 pyproject.toml
"""
[project]
dependencies = [
    "fastapi",  # 没有版本范围
    "some-unreleased-package",  # 拼写错误
]
"""
# 应使用 uv add fastapi 自动管理


# 错误 4：全局安装项目依赖

# 不使用虚拟环境，全局安装
uv pip install fastapi  # 污染全局环境

# 应在项目目录下使用 uv add
cd my-project
uv add fastapi


# 错误 5：忽略 Python 版本兼容性

# pyproject.toml 中声明 requires-python = ">=3.10"
# 但实际使用了 Python 3.13 才有的特性
# 应先在项目中固定 Python 版本
uv python pin 3.11
```

## 工具链配置

```toml
# .python-version — Python 版本文件
3.11

# pyproject.toml uv 相关配置
[tool.uv]
# 默认分组
default-groups = ["dev"]

# 索引配置
[[tool.uv.index]]
name = "pypi"
url = "https://pypi.org/simple/"
default = true

# 缓存的配置（CI 环境下）
# UV_CACHE_DIR=/tmp/uv-cache
# UV_LINK_MODE=copy
```
