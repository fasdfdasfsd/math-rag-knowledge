# 知识反馈闭环协议

## 一、闭环总览

```
┌─────────────────────────────────────────────────────────┐
│ 阶段一：捕获                                             │
│   操作经验 → MEMORY.md 即时写入                           │
│   触发：即时写入条件（决策/bug修复/避坑/阶段完成/PreCompact）│
├─────────────────────────────────────────────────────────┤
│ 阶段二：提醒                                             │
│   SessionStart hook → 输出未归类避坑数量                   │
│   触发：每次会话启动                                       │
├─────────────────────────────────────────────────────────┤
│ 阶段三：归类                                             │
│   人工确认 → knowledge/避坑汇总.md 对应分类                │
│   触发：SessionStart 提醒后用户决定                        │
├─────────────────────────────────────────────────────────┤
│ 阶段四：固化                                             │
│   提炼为规范规则 → knowledge/ 对应分类文件                  │
│   如涉及具体文件路径 → .claude/rules/ 路径规则              │
│   更新规则-强制映射表                                      │
│   触发：避坑经验验证具有通用价值后                           │
├─────────────────────────────────────────────────────────┤
│ 阶段五：审查                                             │
│   季度审查 → 过期知识加 deprecated 标记或归档               │
│   触发：按知识审查协议                                     │
└─────────────────────────────────────────────────────────┘
```

## 二、各阶段执行标准

### 阶段一：捕获

| 要素 | 规范 |
|------|------|
| 触发条件 | 5 个即时写入条件（参见 CLAUDE.md §记忆管理） |
| 写入位置 | `.claude/MEMORY.md` 的「避坑记录」段落 |
| 格式要求 | `序号. <问题描述> → <解决方案>` |
| 责任人 | Claude（自动执行） |

### 阶段二：提醒

| 要素 | 规范 |
|------|------|
| 触发时机 | 每次 SessionStart |
| 输出格式 | `📝 避坑: MEMORY.md 中有 N 条未归类经验` |
| 实现方式 | SessionStart hook（已配置） |

### 阶段三：归类

| 要素 | 规范 |
|------|------|
| 触发方式 | 用户看到 SessionStart 提醒后决定 |
| 归类目标 | `knowledge/避坑汇总.md` 对应分类表格 |
| 记录格式 | `问题 → 原因 → 解决方案 → 日期` |
| 责任人 | 用户 + Claude 协作 |

### 阶段四：固化

| 要素 | 规范 |
|------|------|
| 触发条件 | 避坑经验被验证具有通用价值（出现 ≥2 次或影响 ≥2 人） |
| 固化方式 | 提炼为 spec 规则，加入对应 `knowledge/` 文件 |
| 路径规则 | 如规则与特定文件路径相关，写入 `.claude/rules/` |
| 映射更新 | 新增规则必须更新 `knowledge/规则强制执行映射表.md` |

### 阶段五：审查

| 要素 | 规范 |
|------|------|
| 周期 | 每季度一次 |
| 检查项 | 过期规则、矛盾规则、废弃知识 |
| 操作 | 参见 `knowledge/知识审查协议.md` |

## 三、度量指标

| 指标 | 目标 | 当前 |
|------|------|------|
| 避坑闭环率 | 100%（每条避坑 30 天内归入避坑汇总） | 0%（刚建立） |
| 规范覆盖率 | must 规则 100% 有强制手段 | 23%（5/22） |
| 知识新鲜度 | 无超过 12 月未更新的活跃规范文件 | 待审计 |
| 审查按时率 | 每季度一次，不延期超过 30 天 | 待首次 |
