# 检索管道设计

## 概述

检索管道是 RAG 系统的核心链路。单一路径（如仅向量检索）往往无法覆盖所有检索场景。本规范采用多路召回策略，结合向量检索、BM25 关键词检索和混合检索，并通过查询重写和重排序提升最终检索质量。

## 核心规则

### 🔴 必须遵守 (MUST)

1. **采用多路召回策略**
   - 至少包含向量检索 + 关键词检索两种方式
   - 向量检索负责语义匹配，关键词检索负责精确匹配

2. **检索结果必须经元数据过滤**
   - 按文档来源、日期范围、文档类型等元数据字段预过滤
   - 确保检索结果在业务上下文中是合法的

3. **检索结果必须重排序**
   - 多路召回结果融合后必须进行重排序
   - 重排序前的 top_k 应大于最终需要的数量

### 🟡 强烈建议 (SHOULD)

1. **查询重写**
   - Query2doc：将简短的查询扩展为文档风格描述
   - HyDE：先生成假设性文档，再用其 embedding 检索
   - 多查询生成：生成多个不同角度的查询词

2. **检索参数调优**
   - `top_k`：初始检索 20 条，重排序后保留 5 条
   - `similarity_threshold`：余弦相似度阈值 0.7 以下的结果过滤

3. **检索管道可观测性**
   - 记录每次检索的延迟、召回数量、各路由召回占比
   - 定期评估检索质量指标

### 🟢 可选建议 (MAY)

1. **自适应检索**
   - 根据查询复杂度动态调整 top_k
   - 简单问题减少检索数量，复杂问题增加检索数量

2. **用户反馈闭环**
   - 用户点击/采纳结果作为正反馈信号
   - 改进检索排序算法

## 正确示例

```python
import asyncio
from typing import List
from dataclasses import dataclass
from abc import ABC, abstractmethod


@dataclass
class RetrievalResult:
    content: str
    metadata: dict
    score: float
    source: str  # "vector", "bm25", "hybrid"


class BaseRetriever(ABC):
    """检索器基类"""
    
    @abstractmethod
    async def retrieve(self, query: str, top_k: int = 20) -> List[RetrievalResult]:
        ...


class VectorRetriever(BaseRetriever):
    """向量检索"""
    
    def __init__(self, embedding_service, vector_store):
        self.embedding = embedding_service
        self.vector_store = vector_store
    
    async def retrieve(self, query: str, top_k: int = 20) -> List[RetrievalResult]:
        query_embedding = await self.embedding.embed_query(query)
        results = await self.vector_store.similarity_search(
            query_embedding, top_k
        )
        return [
            RetrievalResult(
                content=r["text"],
                metadata=r["metadata"],
                score=r["similarity"],
                source="vector",
            )
            for r in results
        ]


class BM25Retriever(BaseRetriever):
    """BM25 关键词检索"""
    
    def __init__(self, documents: List[str]):
        from rank_bm25 import BM25Okapi
        import jieba
        
        # 中文分词
        tokenized_corpus = [
            list(jieba.cut(doc)) for doc in documents
        ]
        self.bm25 = BM25Okapi(tokenized_corpus)
        self.documents = documents
    
    async def retrieve(self, query: str, top_k: int = 20) -> List[RetrievalResult]:
        import jieba
        tokenized_query = list(jieba.cut(query))
        scores = self.bm25.get_scores(tokenized_query)
        
        # 获取 top_k
        top_indices = sorted(
            range(len(scores)),
            key=lambda i: scores[i],
            reverse=True
        )[:top_k]
        
        return [
            RetrievalResult(
                content=self.documents[i],
                metadata={"index": i},
                score=scores[i],
                source="bm25",
            )
            for i in top_indices
        ]


class QueryRewriter:
    """查询重写"""
    
    def __init__(self, llm_client):
        self.llm = llm_client
    
    async def hyde(self, query: str) -> str:
        """HyDE: 生成假设性文档"""
        prompt = (
            f"请根据以下问题，写一段假设性的文档段落来回答该问题：\n\n"
            f"问题：{query}\n\n"
            f"假设性回答："
        )
        response = await self.llm.generate(prompt)
        return response
    
    async def multi_query(self, query: str) -> List[str]:
        """生成多个查询角度"""
        prompt = (
            f"原始查询：{query}\n\n"
            f"请从不同角度生成 3 个相关的搜索查询："
        )
        response = await self.llm.generate(prompt)
        queries = [q.strip() for q in response.split("\n") if q.strip()]
        return [query] + queries  # 包含原查询


class RetrievalPipeline:
    """检索管道 — 多路召回 + 过滤 + 重排序"""
    
    def __init__(self, retrievers: List[BaseRetriever]):
        self.retrievers = retrievers
        self.query_rewriter = None
    
    async def retrieve(self, query: str, top_k: int = 20, 
                       final_top_k: int = 5) -> List[RetrievalResult]:
        # Step 1: 查询重写（可选）
        rewritten_query = query
        if self.query_rewriter:
            rewritten_query = await self.query_rewriter.hyde(query)
        
        # Step 2: 多路召回
        all_results: List[RetrievalResult] = []
        for retriever in self.retrievers:
            results = await retriever.retrieve(rewritten_query, top_k)
            all_results.extend(results)
        
        # Step 3: 元数据过滤
        filtered_results = self._metadata_filter(all_results)
        
        # Step 4: RRF 融合（在重排序模块中）
        fused_results = self._reciprocal_rank_fusion(filtered_results)
        
        # Step 5: 返回 top_k
        return fused_results[:final_top_k]
    
    @staticmethod
    def _metadata_filter(results: List[RetrievalResult]) -> List[RetrievalResult]:
        """元数据过滤"""
        return [
            r for r in results
            if r.metadata.get("is_deleted") is not True
        ]
    
    @staticmethod
    def _reciprocal_rank_fusion(results: List[RetrievalResult], k: int = 60) -> List[RetrievalResult]:
        """RRF 融合排序"""
        from collections import defaultdict
        
        # 按内容去重并计算 RRF 分数
        scores: dict[str, float] = defaultdict(float)
        content_map: dict[str, RetrievalResult] = {}
        
        for i, r in enumerate(results):
            key = r.content[:100]  # 用前缀做近似去重
            scores[key] += 1 / (k + i + 1)
            if key not in content_map:
                content_map[key] = r
        
        # 按 RRF 分数排序
        sorted_keys = sorted(scores.keys(), key=lambda k: scores[k], reverse=True)
        return [content_map[k] for k in sorted_keys]
```

## 错误示例

```python
# 错误 1：仅用向量检索
def bad_only_vector():
    """只用向量检索 → 关键词精确匹配失败"""
    
    # 查询："FastAPI 路由装饰器写法"
    # 向量检索可能返回：
    # - "FastAPI 是高性能 Web 框架"
    # - "Django URL 配置"
    # - "Flask 路由注册"
    # 
    # 丢失的关键词匹配：
    # - "@app.get('/path')" 这种代码片段 embedding 效果差
    # - 精确匹配 "route"、"decorator" 等关键词被忽略


# 错误 2：不使用元数据过滤
def bad_no_metadata_filter():
    """不过滤 → 检索到不该出现的内容"""
    
    results = vector_store.search(query, top_k=20)
    # 返回了：
    # - 过期版本的文档（version=1.0，当前是 2.0）
    # - 内部开发笔记（is_internal=true）
    # - 已废弃的 API 文档（status=deprecated）
    # 
    # 应当过滤：version >= 2.0, is_internal=false, status=active


# 错误 3：检索后不重排序
def bad_no_rerank():
    """多路召回后简单拼接，不重排序"""
    
    vector_results = vector_retriever.retrieve(query, top_k=10)
    bm25_results = bm25_retriever.retrieve(query, top_k=10)
    
    # 简单拼接：向量结果在前，关键词结果在后
    # 低质量的向量结果可能排在高质量的关键词结果之前
    final_results = vector_results + bm25_results
    # 应使用 RRF 或 Cross-encoder 融合重排


# 错误 4：top_k 设置不合理
def bad_top_k():
    """一次性检索过多或过少"""
    
    # 检索太少（top_k=2）
    # 可能错过最佳匹配
    results = retriever.retrieve(query, top_k=2)
    
    # 或者检索太多又不重排序（top_k=1000）
    # 大量低质量结果干扰 LLM 生成
    results = retriever.retrieve(query, top_k=1000)
```

## 工具链配置

```python
# 检索管道配置
from dataclasses import dataclass, field

@dataclass
class RetrievalConfig:
    """检索管道配置"""
    # 多路召回
    enable_vector: bool = True
    enable_bm25: bool = True
    enable_hybrid: bool = True
    
    # 检索参数
    vector_top_k: int = 20
    bm25_top_k: int = 20
    similarity_threshold: float = 0.7
    
    # 重排序
    rerank_top_k: int = 5
    enable_cross_encoder: bool = True
    
    # 查询重写
    enable_query_rewrite: bool = True
    rewrite_method: str = "hyde"  # "hyde", "multi_query", "query2doc"
```
