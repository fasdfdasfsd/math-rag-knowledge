# 中国数据安全与个人信息保护

## 概述

随着《中华人民共和国数据安全法》（2021.09.01 施行）和《中华人民共和国个人信息保护法》（PIPL，2021.11.01 施行）的落地，中国构建了以数据分类分级、个人信息保护、跨境数据管理为核心的监管体系。配合网络安全等级保护（等保 2.0），形成了完整的数据安全合规框架。

---

## 核心要点

### 《数据安全法》核心要求

- **数据分类分级制度**（第 21 条）：国家建立数据分类分级保护制度，按数据重要性分为一般数据、重要数据、核心数据
- **数据安全保护义务**（第三章）：
  - 建立数据全生命周期安全管理制度
  - 开展数据安全教育培训
  - 采取必要技术措施防范数据泄露
  - 数据安全风险评估与监测预警
- **数据安全审查**（第 24 条）：对影响国家安全的处理活动进行审查
- **数据出口管制**（第 25 条）：对关键数据实施出口管制
- **违法处罚**（第六章）：最高可处 1000 万元罚款，情节严重可吊销营业执照

### 《个人信息保护法》（PIPL）核心条款

| 条款 | 内容 | 说明 |
|------|------|------|
| 第 4 条 | 个人信息定义 | 已识别或可识别的自然人有关信息（匿名化除外） |
| 第 6 条 | 最小必要原则 | 收集限于实现目的的最少信息 |
| 第 13-17 条 | 告知同意规则 | 显著方式告知，单独同意敏感信息 |
| 第 23 条 | 委托处理 | 委托第三方处理需签署协议并监督 |
| 第 38-43 条 | 跨境传输 | 安全评估 + 认证 + 标准合同三条路径 |
| 第 55 条 | 个人信息保护影响评估 | 处理敏感信息 / 委托处理前必须评估 |
| 第 57 条 | 泄露通知 | 泄露后应立即通知监管与个人 |
| 第 66 条 | 处罚 | 最高 5000 万元或上年营业额 5% |

### 数据分类分级制度

```
核心数据 ── 危害国家安全 → 最高等级保护（访问控制 + 加密 + 审计）
重要数据 ── 危害公共利益 → 高等级保护（脱敏 + 加密 + 跨境审批）
一般数据 ── 危害个人权益 → 基础等级保护（最小必要 + 授权管理）
```

- **重要数据目录**由行业主管部门制定
- 处理重要数据应 **定期开展风险评估**，并向有关部门报送评估报告

### 等保 2.0 技术要求

| 安全层面 | 关键技术措施 |
|----------|-------------|
| 物理安全 | 机房访问控制、温湿度监控、UPS、防雷接地 |
| 网络安全 | 边界防火墙、入侵检测/防御（IDS/IPS）、VPN、流量审计 |
| 主机安全 | 身份鉴别、访问控制、服务器安全基线、主机入侵检测 |
| 应用安全 | 应用层身份认证、权限控制、防 SQL 注入/XSS、日志审计 |
| 数据安全 | 数据分类分级、数据加密（传输 + 存储）、备份恢复、数据脱敏 |

- 二级系统：每年一次安全评估；三级系统：每半年一次
- 三级系统要求 **密码技术** 实现数据完整性与保密性

## 实际合规检查清单

### 开发阶段

- [ ] 评估处理的数据类型与级别（一般/重要/核心数据）
- [ ] 设计阶段嵌入隐私保护（Privacy by Design）
- [ ] 最小必要原则：只采集业务必需的个人信息
- [ ] 数据加密方案：传输层 TLS 1.2+，存储层 AES-256

### 运行阶段

- [ ] 个人信息保护影响评估（PIPL 第 55 条）已执行
- [ ] 数据安全管理制度已发布并培训
- [ ] 日志系统记录数据访问操作（谁、何时、何地、何种数据）
- [ ] 数据泄露应急响应预案已建立
- [ ] 用户行使删除权/更正权的流程已实现

### 合规检查

- [ ] 等保等级已定级并通过测评
- [ ] 数据跨境传输已完成安全评估或标准合同备案
- [ ] 隐私政策（Privacy Policy）符合告知同意要求
- [ ] 数据处理协议（DPA）已与第三方签署
- [ ] 年度数据安全风险评估已完成
