# 中国 GB/T 标准体系

## 概述

中国国家标准（GB/T 推荐性标准）在软件工程领域形成了完整的体系，涵盖软件生命周期过程、质量度量与评价、网络安全等级保护以及需求/测试文档规范。多数 GB/T 标准为 ISO/IEC 标准的采用或等效转化。

---

## 核心要点

### 1. GB/T 8566-2022 — 软件生存周期过程

- **修改采用 ISO/IEC 12207:2017**，替代 GB/T 8566-2007
- 定义了 4 大类过程：

| 过程类别 | 包含过程数 | 说明 |
|----------|-----------|------|
| 协议过程 | 2 | 获取、供应 |
| 组织级使能过程 | 6 | 管理、基础设施、改进、人力资源、质量、复用 |
| 技术管理过程 | 8 | 策划、评估、风险、配置、决策、测量等 |
| 技术过程 | 12 | 需求分析、设计、构建、集成、测试、部署等 |

- 新增了 **敏捷开发** 和 **DevOps** 相关实践指引
- 适用于软件产品开发、服务和获取的组织

### 2. GB/T 25000 系列 — 质量测量与评价

基于 ISO/IEC 25000 系列（SQuaRE），包含：

| 标准号 | 内容 |
|--------|------|
| GB/T 25000.1 | 质量模型框架与概述 |
| GB/T 25000.10 | 系统与软件产品质量模型（= ISO 25010） |
| GB/T 25000.20 | 质量测量框架（Quality Measurement Framework） |
| GB/T 25000.21 | 质量度量元素（Quality Measure Elements） |
| GB/T 25000.30 | 质量需求框架（Quality Requirements Framework） |
| GB/T 25000.40 | 评价过程（Evaluation Process） |
| GB/T 25000.41 | 评价者指南 |
| GB/T 25000.51 | 就绪可用软件产品的质量要求和测试 |

- **质量度量周期**：度量需求 → 度量定义 → 度量执行 → 评价结果
- 定义了 **质量度量元素（QME）** 作为最小可度量单元

### 3. GB/T 22239-2019 — 网络安全等级保护（等保 2.0）

2019 年 12 月 1 日正式实施，核心变化：

- **保护范围扩展**：从信息系统扩展到网络基础设施、云计算、移动互联、物联网、工业控制系统
- **安全要求分级**：一级（最低）到四级（最高）
- **技术要求分类**：
  - 物理安全
  - 网络安全
  - 主机安全
  - 应用安全
  - 数据安全与备份恢复
- **管理要求分类**：
  - 安全管理制度
  - 安全管理机构
  - 人员安全管理
  - 系统建设管理
  - 系统运维管理

### 4. GB/T 9385 / 9386 — 需求与测试文档

| 标准 | 内容 | 对应国际标准 |
|------|------|-------------|
| GB/T 9385-2008 | 软件需求规格说明（SRS）规范 | 等效采用 IEEE 830 |
| GB/T 9386-2008 | 软件测试文档规范 | 等效采用 IEEE 829 |

- GB/T 9385 定义了 SRS 的章节结构和编写要求
- GB/T 9386 覆盖测试计划、测试说明、测试报告三类文档

### 国家标准与国际标准的对应关系

| 中国标准 | 对应国际标准 | 关系类型 |
|----------|-------------|----------|
| GB/T 8566-2022 | ISO/IEC 12207:2017 | 修改采用（MOD） |
| GB/T 25000 系列 | ISO/IEC 25000 系列 | 等同采用（IDT） |
| GB/T 22239-2019 | 自主制定（参考 ISO 27001） | 中国特色标准 |
| GB/T 9385-2008 | IEEE 830 | 等效采用 |
| GB/T 9386-2008 | IEEE 829 | 等效采用 |
| GB/T 19001-2016 | ISO 9001:2015 | 等同采用（IDT） |

## 实际应用

1. **软件公司建立过程体系** → 参考 GB/T 8566 定义组织级软件过程
2. **产品质量度量** → 采用 GB/T 25000.10 质量模型 + GB/T 25000.21 度量元素
3. **政府/国企项目投标** → 项目需满足 GB/T 22239（等保）相应等级要求
4. **编写需求文档** → 按 GB/T 9385 模板组织 SRS
5. **编写测试文档** → 按 GB/T 9386 组织测试计划与报告
