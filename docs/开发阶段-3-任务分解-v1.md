## 0. 开发者提交前自检命令

> 在每次提交前执行以下完整序列。全部通过后才可 push。

```bash
# ═══════════════════════════════════════════════
# 环境准备
# ═══════════════════════════════════════════════
# 确保安装了项目依赖
uv sync --group dev

# 设置 PYTHONPATH（确保 src 可被 mypy/pytest 正确识别）
export PYTHONPATH=src:$PYTHONPATH

# ═══════════════════════════════════════════════
# 1. 格式化代码
# ═══════════════════════════════════════════════
echo "=== 1/5: ruff format ==="
uv run ruff format src/ tests/

# ═══════════════════════════════════════════════
# 2. Lint 检查
# ═══════════════════════════════════════════════
echo "=== 2/5: ruff check ==="
uv run ruff check src/ tests/ || exit 1

# ═══════════════════════════════════════════════
# 3. 类型检查（strict 模式）
# ═══════════════════════════════════════════════
echo "=== 3/5: mypy strict ==="
uv run mypy --strict src/ || exit 1

# ═══════════════════════════════════════════════
# 4. 测试 + 覆盖率
# ═══════════════════════════════════════════════
echo "=== 4/5: pytest + coverage ==="
uv run pytest \
  --cov=src \
  --cov-report=term \
  --cov-report=html:htmlcov \
  --cov-fail-under=80 \
  tests/ || exit 1

# 核心决策引擎独立覆盖率
echo "=== 4b/5: decision engine coverage ==="
uv run pytest \
  --cov=src/services/decision_engine \
  --cov-report=term \
  --cov-fail-under=90 \
  tests/services/decision_engine/ || exit 1

# ═══════════════════════════════════════════════
# 5. 依赖安全检查
# ═══════════════════════════════════════════════
echo "=== 5/5: pip-audit ==="
uv run pip-audit --strict -r pyproject.toml || exit 1

# ═══════════════════════════════════════════════
echo "=== 全部通过 — 可以提交 ==="
```

> **小技巧**: 可将上述命令保存为 `scripts/precommit.sh`，然后 `chmod +x scripts/precommit.sh && ./scripts/precommit.sh` 一键执行。

**交付检查**：

## 4. 复杂度汇总

| 1. 基础设施 | 3 | 低 | 低 — 标准化搭建 |
| 2. 数据模型 | 2 | 低-中 | 低 |
| 3. 决策引擎 | 3 | **高** | 中 — 算法参数需要调优 |
| 4. RAG 约束层 | 2.5 | 中 | 中 — 递归 CTE 性能 |
| 5. LLM 生成层 | 2.5 | **高** | **高** — SSE 流式 + Prompt 质量 |
| 6. 内容校验 | 1.5 | 中 | 低 — 规则引擎成熟技术 |
| 7. Background Worker | 1.5 | 中 | 低 |
| 8. API 路由 | 2 | 中 | 低 |
| 9. 前端 | 18.5 | **高** | **高** — 六段式叙事UI复杂度 |

### 风险应对

| 风险 | 概率 | 影响 | 应对 |
|------|:----:|:----:|------|
| LLM Prompt 质量不达预期 | 中 | 高 | 准备 3 版 Prompt 迭代 + 安全模板兜底 |
| SSE 流式分割与校验延迟 | 中 | 中 | 首段 3s 切片推送，后续段后台校验 |
| 递归 CTE 查询超 50ms | 低 | 中 | 加物化路径列冗余，避免深度递归 |
| 预生成池命中率不足 60% | 中 | 中 | 预热策略 + 典型画像模板池 |
| 前端六段式渲染性能 | 低 | 高 | 虚拟列表 + 分段加载 |
| Django-style N+1 查询 | 中 | 中 | Repository 层强制 eager load / 批量查询 |
