## 一、原始需求回顾

### 产品愿景

> 千人千面、无限更新、永不结局、专属数学博士陪伴的数学冒险世界。

### 原始技术方案要点

| 层级 | 内容 |
|------|------|
| 应用交互层 | 前端沉浸式交互、剧情展示、闯关对话、音色播报 |
| AI 调度层 | LLM 调用、Prompt 工程、剧情生成、数学博士人设管控 |
| RAG 知识库层 | 公共课本知识库 + 用户私有学情库，双库隔离 |
| 数据持久层 | 实时存储学习行为、更新用户画像 |

### 原始方案的核心链路

```
RAG 检索 → LLM 决定考什么 + 生成关卡 → LLM 判定对错 → 数据回写
```

### 2.1 《神奇校车》The Magic School Bus

**核心模式：向导带领 + 载具变形 + 体验式学习**

```
┌────────────────────────────────────────────┐
│  《神奇校车》叙事公式                         │
│                                            │
│  教室（安全起点）                            │
│     ↓                                      │
│  弗瑞丝老师提出"疯狂问题"（触发好奇）          │
│     ↓                                      │
│  校车变形（缩小/飞天/潜水）→ 进入异世界        │
│     ↓                                      │
│  孩子们亲历科学现象（体验即学习）              │
│     ↓                                      │
│  遭遇意外/危机（冲突驱动）                    │
│     ↓                                      │
│  用科学知识解决问题（知识 = 超能力）           │
│     ↓                                      │
│  返回教室 + 总结（安全闭环）                  │
└────────────────────────────────────────────┘
```

| 叙事元素 | 设计 | 对我们的借鉴 |
|----------|------|-------------|
| **向导角色** | 弗瑞丝老师 — 永远不直接给答案，只说"去看看吧" | 数学博士 = 弗瑞丝式引导者，不报答案，带学生"进数学里面看看" |
| **载具变形** | 校车 → 缩小进血管、变形潜水艇、飞天巴士 | 学生的"冒险载具"随世界主题变形（魔法扫帚→潜水器→星际飞船） |
| **体验即学习** | 孩子不是"被教"，而是亲眼看到/亲手摸到 | 数学概念应该"看得见"（分数不是被教，而是进入了分数之城看到一切被切开） |
| **安全闭环** | 每次冒险结束回到教室，强化"探索可归来"的安全感 | 每关结束回到"基地/家园"，博士总结，学生获得新装备/称号 |
| **容错口号** | "Take chances, make mistakes, get messy!" | 系统需要明确的"鼓励犯错"信号，答错不是失败是探索的一部分 |

### 2.2 《神奇树屋》Magic Tree House

**核心模式：传送门 + 任务驱动 + 时间空间无限**

```
┌────────────────────────────────────────────┐
│  《神奇树屋》叙事公式                         │
│                                            │
│  发现线索（树屋里的书/物品）                  │
│     ↓                                      │
│  说出愿望 → 树屋旋转 → 时空传送              │
│     ↓                                      │
│  到达新世界（恐龙时代/中世纪/月球…）           │
│     ↓                                      │
│  摩根给出任务（找到某物/帮助某人/解开谜题）     │
│     ↓                                      │
│  探索中遇到困难 → 用知识+观察力解决           │
│     ↓                                      │
│  完成任务 → 获得新物品/线索 → 返回树屋         │
│     ↓                                      │
│  新物品指向下一个冒险（钩子）                  │
└────────────────────────────────────────────┘
```

| 叙事元素 | 设计 | 对我们的借鉴 |
|----------|------|-------------|
| **传送门机制** | 树屋 = 无限世界的入口，每本书 = 一个新目的地 | 学生的"传送门"每次打开随机新地图，地图间有线索链 |
| **任务委托方** | 摩根·勒菲 — 神秘图书管理员，给出任务和奖励 | 数学博士既是同伴也是"任务委托者"，需要学生帮忙解决数学危机 |
| **Fact Tracker** | 配套非虚构读物，把故事里的知识单列出来 | 每个冒险关卡配套"知识卡片"（课后可复习的知识点总结） |
| **钩子设计** | 每次冒险结尾的新物品/线索 → 自然引发下次冒险的欲望 | 关卡结束必有"悬念钩子"：地图上亮起新区域、博士收到新求救信号 |
| **兄妹双主角** | Jack（理性/笔记/分析）+ Annie（直觉/勇敢/行动） | 两种思维模式对应两种解题风格，系统可以识别学生类型并适配引导方式 |

### 2.3 李毓佩数学故事

**核心模式：数学概念拟人化 + 冲突即知识 + 双线叙事**

```
┌────────────────────────────────────────────┐
│  李毓佩数学故事叙事公式                        │
│                                            │
│  主角误入数学王国（传送/梦境/奇遇）            │
│     ↓                                      │
│  遭遇拟人化的数学概念（零国王/分数王子/小数精灵） │
│     ↓                                      │
│  数学概念之间的"战争/冲突" = 知识本身的矛盾     │
│     ↓                                      │
│  主角必须"破案/调解/战斗"→ 解题推动情节       │
│     ↓                                      │
│  数学逻辑线（怎样解题）↔ 文学叙事线（怎样过关）  │
│     ↓                                      │
│  胜利 = 理解了这个数学概念                     │
│     ↓                                      │
│  获得新能力/盟友 → 进入下一个数学领域          │
└────────────────────────────────────────────┘
```

| 叙事元素 | 设计 | 对我们的借鉴 |
|----------|------|-------------|
| **数学概念拟人化** | 零国王（矛盾协调者）、分数王子、小数点精灵（调皮） | 核心数学概念需要设计为有性格的"角色"，学生不是做题而是"和分数王子交朋友" |
| **冲突即知识** | 奇偶数之战、真分数假分数之争 — 矛盾本身就是知识 | 把知识难点设计为戏剧冲突：不是"学习分数比较"，而是"调解分数王国内战" |
| **双线叙事** | 故事线 + 数学逻辑线，彼此推动，有机融合 | LLM Prompt 必须同时输出"剧情文本"和"数学推导链"，两条线缺一不可 |
| **疑案代替应用题** | 不是"请计算 3/4 + 1/2"，而是"分数王国发生了食物失踪案，3/4块蛋糕和1/2块蛋糕到底谁拿了谁的" | 每道题必须有"案件包装"，学生是"数学侦探" |
| **问题链自然引出** | 提问者不固定（主角、配角、"龙套"均可），提问方式多样 | 提示和引导不应总是博士给出，路边的NPC、环境提示都可以抛出问题 |
| **减少术语，增加意象** | "分数线" → "双层宝座的隔板"；"四则运算" → "四把钩子" | 低年级学生的关卡中，数学术语必须"翻译"为具象意象 |

### 2.4 世界旅行奇遇

**核心模式：地点即课程 + 文化沉浸 + 收藏驱动**

| 叙事元素 | 设计 | 对我们的借鉴 |
|----------|------|-------------|
| **地点即课程** | 每到一个国家/城市，学习当地的数学史/建筑几何/文化数学 | 地图节点 = 真实的世界地点，埃及金字塔（几何）、荷兰风车（比例）、中国长城（测量） |
| **护照/收藏** | 每完成一个地点获得印章、纪念品 | "数学护照"盖章系统，收集驱动持续回归 |
| **历史数学家彩蛋** | 遇见少年毕达哥拉斯、小祖冲之、年轻的阿基米德 | NPC 角色可以嵌入数学史上真实人物（作为"同伴"或"委托人"） |
| **建筑揭示数学** | 当地建筑/文化符号中隐藏数学原理 | 关卡场景可以基于真实地标，在探索中自然发现数学 |

### 2.6 叙事设计指导原则

| 原则 | 出处 | 含义 |
|------|------|------|
| **知识不是教的，是遇见的** | 神奇校车 | 数学概念应该作为"世界里的存在"被学生发现，而非被告知 |
| **冲突驱动，而非习题驱动** | 李毓佩 | 每关的核心不是"做几道题"，而是"解决一个数学引发的危机" |
| **永远有下一扇门** | 神奇树屋 | 关卡结束必有新线索，形成自然的"再来一次"驱动力 |
| **数学是超能力，不是任务** | 融合 | 学生的数学能力是推进剧情的武器，用分数知识打败分数怪物 |
| **安全探索，勇敢犯错** | 神奇校车 | 系统设计必须让"答错"不带来惩罚感，而是探索的一部分 |
| **角色是有记忆的** | 融合 | NPC 记住学生的选择、偏好、成就，形成持续的情感连接 |

## 三、行业对标关键发现

### 3.1 Duolingo 双层解耦架构（最核心参考）

Duolingo 的架构刻意分为两层，LLM 并非全链路决策者：

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│  Birdbrain（传统 ML，成本极低）   GPT-4（LLM，成本高）  │
│  ┌──────────────────────────┐  ┌──────────────────┐  │
│  │ "答什么、何时答"           │  │ "怎么呈现"        │  │
│  │ · 知识掌握度追踪           │  │ · 对话生成        │  │
│  │ · 遗忘曲线预测             │  │ · 解释生成        │  │
│  │ · 难度匹配                 │  │ · 角色扮演        │  │
│  │ · 间隔复习调度             │  │ · 内容包装        │  │
│  └──────────────────────────┘  └──────────────────┘  │
│      确定性算法                      创造性生成        │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**核心启示**：能用算法解决的不用 LLM，能用检索解决的不用生成，LLM 只做最擅长的事——内容创作和角色扮演。

## 四、核心架构优化：从"LLM 全链路"到"三层解耦"

### 原始方案的问题

LLM 被放在所有决策链路的核心位置，但 LLM **不擅长精确判断**（该考什么、难度多少），只擅长内容创作。

### 优化后的三层架构

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  Layer 1: 决策引擎（确定性算法，零 LLM 成本）              │
│  ┌─────────────────────────────────────────────────┐   │
│  │ · 知识点选择：掌握度矩阵 + 遗忘曲线 + 进度状态机     │   │
│  │ · 难度调节：IRT 项目反应理论 / BKT 贝叶斯知识追踪    │   │
│  │ · 排重逻辑：历史标签哈希 → O(1) 查重               │   │
│  │ · 对错判定：规则引擎（数学题答案可确定性验证）        │   │
│  │ · 进度控制：有限状态机（FSA），不靠 LLM              │   │
│  └─────────────────────────────────────────────────┘   │
│                         ↓                               │
│  Layer 2: RAG 约束层（检索，不用 LLM 决策）               │
│  ┌─────────────────────────────────────────────────┐   │
│  │ · 公共库检索：锚定知识点、课标要求、易错点           │   │
│  │ · 私有库检索：上次错题、薄弱环节、偏好标签、历史剧情   │   │
│  │ · KG 增强：前置依赖知识点、关联扩展知识点            │   │
│  │ · 输出：结构化的"关卡约束包"（JSON）                │   │
│  └─────────────────────────────────────────────────┘   │
│                         ↓                               │
│  Layer 3: LLM 生成层（唯一使用 LLM 的地方）              │
│  ┌─────────────────────────────────────────────────┐   │
│  │ · 输入：Layer 1 的决策参数 + Layer 2 的约束包       │   │
│  │ · 输出：沉浸式剧情文本、数学博士对话、引导话术        │   │
│  │ · 约束：固定人设 Prompt + 输出格式模板 + 后置校验     │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 分工原则

| 任务类型 | 用什么 | 原因 |
|----------|--------|------|
| 该考哪个知识点 | 算法（掌握度矩阵） | LLM 选择不可靠、不可解释 |
| 该出多难的题 | 算法（IRT/BKT） | 需要精确数值计算 |
| 这道题答对了吗 | 规则引擎 | 数学答案可确定性验证 |
| 剧情怎么讲 | LLM | 这是 LLM 最擅长的 |
| 博士怎么引导 | LLM | 需要自然语言生成 |
| 内容有没有超纲 | KG + 规则 | 需要确定性判定 |

## 五、优化点详解

### 优化 1：难度自适应算法（替换 LLM 判断）

```
Birdbrain 启发 → 我们的难度引擎：

关卡难度 = f(
    知识点掌握率,        // 来自历史答题正确率（权重 0.4）
    遗忘曲线衰减系数,     // 艾宾浩斯模型（权重 0.2）
    最近情绪状态,         // 连续错误 → 降难度保护信心（权重 0.2）
    冒险进度因子,         // 不得跨越未解锁地图（权重 0.2）
)

目标区间：P(正确) ∈ [0.5, 0.7] → "甜蜜点"，既不沮丧也不无聊
```

### 优化 2：知识图谱增强 RAG（替换纯向量检索）

```
纯向量 RAG 问题：检索到的内容碎片化，缺乏逻辑关联

KG 增强方案：
┌──────────────────────────────────────────┐
│  知识点关系图（轻量 KG，PostgreSQL 即可）    │
│                                          │
│  分数加减 ──[前置]──→ 分数乘除              │
│     │                    │               │
│     └──[关联]──→ 小数转换  │               │
│                    │     │               │
│  通分约分 ←──[前置]──┘     │               │
│                            │               │
│  地图A ──[包含]──→ 关卡1 ──[考察]──→ 分数加减 │
│                                          │
└──────────────────────────────────────────┘

每次检索时：
  1. 向量检索 → 语义相似的知识点
  2. KG 查询 → 前置依赖（确保学生已掌握）
            → 后续扩展（为下一关铺垫）
            → 关联易错点（针对薄弱环节）
  3. 合并结果 → 结构化的"关卡约束包"
```

### 优化 7：叙事引擎 — 将四大作品模式工程化

将 2.5 节的统一叙事模板固化为可编程的 Prompt 结构：

```
LLM 关卡生成 Prompt 结构（六段式）：

[SYSTEM] 你是数学冒险世界的叙事引擎。遵循以下模板生成关卡。

[SECTION 1: 入口]
- 随机选择传送门开启方式（树屋旋转/校车变形/魔法书翻开/星门亮起）
- 目标世界预告（今日地点 + 感官描写 3 句）
- 博士引出悬念："听说 [地点] 的 [数学概念] 出了怪事…"

[SECTION 2: 到达]
- 载具着陆变形描写（入水→潜水器，升空→飞船，入地→钻探机）
- 场景展开（视觉 2 句 + 听觉 1 句 + 触觉/嗅觉 1 句）
- 弗瑞丝式引导提问（不报答案，只说"你注意到了吗？"）

[SECTION 3: 冲突 — 使用李毓佩拟人化手法]
- 数学概念呈现为有性格的角色（分数王子傲慢/小数精灵调皮/零国王沉默）
- 冲突包装：战争/失踪/失衡/争吵/诅咒
- 学生的任务 = 解决冲突（而非"做题"）

[SECTION 4: 解题 — 双线叙事]
- 文学线：[剧情推进 2-3 句] → 遇到数学障碍
- 数学线：[引导解题，分步提示] → 学生输入答案
- 文学线：[根据答案推进剧情] → 遇到下一个数学障碍
- （重复 2-4 轮，每关 3-5 道题）

[SECTION 5: 胜利]
- 数学概念恢复和谐（分数王子和解/图形拼图完整/等式平衡）
- 博士情感肯定："你刚刚做了很了不起的事——你用 [概念] 拯救了 [世界]"
- 获得奖励：印章 + 装备碎片 + 新盟友登场

[SECTION 6: 钩子]
- 新线索出现：地图新区域亮起 / 博士收到新求救 / 神秘物品掉落
- 悬念尾句："但是…远处黑暗中好像有什么在动…"
- 明日预告："明天，博士在 [新地点] 等你！"

[EXCLUDE] 禁止的表述：
- "请计算" "正确答案是" "你做错了" "再试一次"
- 代替为："你觉得呢？" "我们来看看发生了什么…" "哇，这个发现太有意思了！"
```

| 叙事工程化参数 | 说明 |
|---------------|------|
| **地点池** | 按年级映射：低年级（魔法森林/海底王国）→ 高年级（星际空间站/古文明遗迹） |
| **载具池** | 随地图主题切换：魔法扫帚/潜水器/星际飞船/时光机/钻探车 |
| **冲突类型** | 战争/失踪案/失衡危机/诅咒破解/谜题寻宝/调解争端 |
| **数学角色池** | 按概念类型预设：数（零国王/分数王子）→ 形（三角形骑士/圆公主）→ 量（时间老人/长度精灵） |
| **感官描写库** | 按场景类型预设形容词/比喻短语，保证每个场景有新鲜的感官体验 |
| **钩子类型** | 求救信号/神秘物品/新地图/失踪人物/预言/季节事件 |

### 优化 3：三种闯关模式轮换 + 叙事赋能（参考四大作品）

| 模式 | 叙事参考 | 设计 | 心理学原理 |
|------|----------|------|-----------|
| **英雄模式** | 李毓佩"疑案破案" | 数学世界出现危机，学生解题解救博士/修复世界 | 使命感 + 英雄叙事 |
| **导师模式** | 神奇树屋"杰克教安妮" | 年轻学徒/小动物向你请教，你"教"会它数学知识 | Protégé Effect（教别人学得更深 1.56×） |
| **探索模式** | 神奇校车"进入看看" | 自由探索地图，发现隐藏的数学彩蛋/历史数学家故事 | 好奇心 + 自主发现驱动 |

三模式按 3:1:1 比例轮换，每完成大地图强制切换。每种模式都遵循 2.5 节的统一叙事模板（入口→到达→冲突→解题→胜利→钩子）。

### 优化 4：世界观持续进化（替换一次生成固定世界观）

```
原方案：初始化时生成世界观 → 固定不变

优化后：
  阶段 1（新手村）：  单一主题世界（如"魔法森林"），入门引导
  阶段 2（世界扩展）：通过大地图后，世界观进化
                    （魔法森林 → 海底王国 → 星际空间站 → 古文明遗迹）
  阶段 3（多元交叉）：历史剧情中的"蝴蝶效应"影响新世界形态
  阶段 4（季节活动）：周期性限时活动（暑假海岛探险 / 寒假冰雪王国）
```

每次世界观切换 = 新的视觉风格 + 新的剧情背景 + 新的角色登场 + 旧知识在新场景中复用。

### 优化 5：模型分级与成本控制

参考 Duolingo 的经验，LLM 按频次分级调用：

| 调用场景 | 频率 | 模型 | 成本 |
|----------|:--:|------|------|
| 关卡剧情生成 | 低频（1 次/天） | deepseek-v4-pro | 高 |
| 博士对话引导 | 中频（每次交互） | deepseek-v4-flash | 中 |
| 课程大纲生成 | 超低频（1 次/学期） | deepseek-v4-pro | 低 |
| 对错判定 | 高频（每次答题） | 规则引擎 | **零** |
| 知识点选择 | 高频（每关） | 算法计算 | **零** |
| 排重检查 | 高频（每关） | 哈希索引 | **零** |
| Embedding | 低频（内容入库时） | DeepSeek Embedding | 低 |

**预估**：相比原始"全链路 LLM"方案，成本降低 **60-70%**。

### 优化 6：题库 + LLM 混合内容策略

| 内容类型 | 生成方式 | 理由 |
|----------|----------|------|
| 数学题目题干 | 人工审核题库 + 参数化模板 | ✅ 保证正确性 |
| 题目数值 | 算法随机生成（约束范围内） | ✅ 无限变体 |
| 解题步骤讲解 | LLM 生成 + 后置校验 | ⚠️ LLM 擅长但需校验 |
| 剧情包装文本 | LLM 生成 | ✅ 纯创意 |
| 角色对话 | LLM 生成 | ✅ 纯创意 |
| 知识点教学顺序 | 课标映射表（静态） | ✅ 确定性 |
| 难度分级 | 算法计算（IRT） | ✅ 精确可控 |
