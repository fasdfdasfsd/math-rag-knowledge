### 1.4 项目范围

**In Scope**：
- 面向 6-14 岁学生的数学冒险游戏化学习平台
- 六段式关卡叙事引擎（入口→到达→冲突→解题→胜利→钩子）
- 三层解耦架构（决策引擎 / RAG 约束 / LLM 生成）
- 双向量库（公共知识库 + 用户私有学情库）
- 数学概念 NPC 角色系统
- 自适应难度调节（IRT 算法驱动）
- 长期记忆与用户画像

**Out of Scope (v1.0)**：
- 多人在线协作/竞技模式
- 家长端独立 App
- 学校/SaaS 管理后台
- 离线模式
- 非数学学科扩展
- 硬件学习机适配

## 2. 总体描述

### 2.1 产品视角

#### 2.1.1 产品定位

"数学冒险世界"是一个 **AI 驱动的游戏化数学学习平台**，与全球 23 款竞品的根本差异在于：

> 每个孩子拥有一个 **专属的、无限更新的、永不结局的数学冒险世界**——这是竞品矩阵中所有现有产品都做不到的。

**核心差异点（来自竞品分析 6 条 DNA）**：

| DNA | 我们的实现 | 竞品对标 |
|-----|-----------|----------|
| 答题必须有后果 | 答题 = 推动冒险剧情，不解谜无法前行的世界 | Prodigy（打怪）/ ST Math（企鹅通行） |
| 错误不能是惩罚 | 答错 → "换个角度前进"，给台阶不打回原形 | Zearn（同级别变式）/ DragonBox（不出现✗） |
| 长线成长有空间 | RAG+LLM 无限生成新关卡、新地图、新角色 | 无竞品实现（均为固定内容库） |
| 角色灵魂共鸣 | 数学概念 NPC 化（零国王/分数王子/三角形骑士） | Numberblocks（数字拟人）/ Mathseeds（宠物系统） |
| 零语言友好 | 大量动画/视觉呈现，降低文字阅读门槛 | ST Math（纯视觉）/ Funexpected Math（零阅读） |
| 学术背书 | IRT 自适应 + 学习数据报告，给家长可量化成果 | Matific（哈佛/斯坦福背书）、DreamBox（过程分析） |

#### 2.1.2 产品架构概览

```
┌────────────────────────────────────────────────────────────┐
│                     应用交互层                               │
│  前端 SPA → 沉浸式冒险画面 → 对话交互 → 答题 → 收藏/成就展示   │
├────────────────────────────────────────────────────────────┤
│                     AI 调度层                                │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│  │ 决策引擎 (L1) │  │ RAG约束 (L2) │  │ LLM生成 (L3) │      │
│  │ 算法驱动      │  │ 双库检索+KG   │  │ 叙事+对话     │      │
│  │ 零LLM成本     │  │ 结构化约束包  │  │ 固定人设Prompt│      │
│  └──────────────┘  └──────────────┘  └──────────────┘      │
├────────────────────────────────────────────────────────────┤
│                     数据持久层                               │
│  用户画像 → 行为日志 → 记忆模块 → 知识图谱 → 向量库 → 题库    │
└────────────────────────────────────────────────────────────┘
```

### 2.2 用户特征

| 用户类型 | 年龄 | 特征 | 核心需求 |
|----------|------|------|----------|
| **主要学习者** | 6-14 岁 | 注意力短（<15 min/关）、动机驱动、对角色有情感依赖 | 好玩 > 学习，想要归属感和成就感 |
| **低年级段** | 6-9 岁 | 识字量有限、具象思维为主、需要更多视觉引导 | 零语言/低语言交互，大量动画 |
| **高年级段** | 10-14 岁 | 抽象思维萌芽、有自学能力、开始关注排名 | 适度文本交互，更深剧情 |
| **家长** | 30-45 岁 | 关注学习效果、愿意为教育付费、需要数据说服 | 可量化进步指标、安全可控 |
| **教师（v2）** | — | 需要班级管理、作业布置 | 仪表盘、班级报告（Out of Scope v1） |

## 3. 外部接口需求

### 3.1 用户界面

| 界面区域 | 功能 | 关键约束 |
|----------|------|----------|
| **冒险地图** | 世界选择、关卡入口、进度可视化 | 地图节点按解锁状态呈现（锁定/可进入/已完成） |
| **关卡对话区** | 博士 NPC 对话、剧情文本、选项交互 | 支持图文混排、LaTeX 公式渲染 |
| **答题交互区** | 题目展示、输入控件、反馈动画 | 答对/答错动画差异化，错误不给红色✗标记 |
| **收藏室/基地** | 装备展示、印章收藏、角色图鉴、知识卡片 | 驱动长线回归的核心界面 |
| **家长报告** | 学习数据、进度摘要、知识点掌握度 | 数据可视化，非技术家长可理解 |

**UI 设计原则**（来自竞品 DNA）：
- **零语言友好**：低年级段界面尽量减少文字说明，用图标/动画/语音引导
- **错误无惩罚**：答错的视觉反馈是"有趣的反弹"而非"红色的惩罚"，参考 DragonBox 和 ST Math
- **即时反馈 < 3 秒**：任何操作 3 秒内必须有视觉/听觉响应，超时触发保底动画

### 3.2 软件接口

| 接口 | 协议 | 用途 |
|------|------|------|
| DeepSeek Chat API | HTTPS / OpenAI-compatible SDK | 关卡剧情生成、博士对话 |
| DeepSeek Embedding API | HTTPS / OpenAI-compatible SDK | 文本向量化 |
| Milvus | gRPC | 向量存储与检索 |
| PostgreSQL | TCP / SQLAlchemy async | 关系数据存储 |
| Redis | TCP / redis-py async | 会话缓存、记忆缓存 |

### 3.3 通信接口

| 通信方式 | 用途 |
|----------|------|
| REST API (FastAPI) | 前端 ↔ 后端所有业务接口 |
| WebSocket (可选 v2) | 实时对话流式输出 |
| HTTPS (TLS 1.3) | 全链路加密传输 |

## 4. 系统功能需求

> 用户故事格式：`As a <角色>, I want <功能>, So that <价值>`
> 验收标准格式：`Given... When... Then...`

### 4.1 模块一：用户与画像系统

#### REQ-USER-001：用户注册与登录

**优先级**：Must have

**用户故事**：
> US-USER-001: As a 新用户（家长代理），I want 用邮箱注册并为孩子创建学习账号，So that 孩子可以开始数学冒险。

**验收标准**：
- AC-USER-001-1 (正常注册): Given 家长输入有效邮箱和密码，When 提交注册表单，Then 系统创建家长账号 + 孩子学习账号，发送验证邮件，引导进入初始水平设定。
- AC-USER-001-2 (密码弱): Given 家长输入弱密码（<8位），When 提交注册表单，Then 系统提示密码强度不足，要求包含大小写字母+数字。
- AC-USER-001-3 (邮箱已注册): Given 邮箱已被注册，When 提交注册表单，Then 系统返回"该邮箱已注册，请直接登录"提示（不区分用户存在与否，安全考虑）。

#### REQ-USER-002：初始水平设定

**优先级**：Must have

**用户故事**：
> US-USER-002: As a 家长，I want 在注册后设定孩子的年级和数学水平，So that 系统能给孩子匹配合适难度的冒险起点。

**验收标准**：
- AC-USER-002-1: Given 注册完成后，When 进入水平设定页，Then 系统提供年级选择（1-9）+ 可选的水平诊断测试（10 道自适应题）。
- AC-USER-002-2: Given 家长跳过诊断测试直接选年级，When 确认，Then 系统以该年级下限难度初始化用户画像，标记为"待校准"。

#### REQ-USER-003：用户画像持续更新

**优先级**：Must have

**用户故事**：
> US-USER-003: As a 系统（决策引擎），I want 每次答题后自动更新用户的知识点掌握度矩阵，So that 下次关卡生成的难度和内容能精准适配学生当前水平。

**验收标准**：
- AC-USER-003-1: Given 学生完成一道题，When 答题结果产生（对/错/部分对），Then 系统在 1 秒内更新对应知识点的掌握度（IRT θ 值）和遗忘曲线参数。
- AC-USER-003-2: Given 学生在某知识点连续答错 3 次，When 第 3 次答错，Then 系统标记该知识点为"困难"，下一关降低该知识点难度并增加视觉辅助。

### 4.2 模块二：冒险世界与关卡系统

#### REQ-WORLD-001：冒险地图

**优先级**：Must have

**用户故事**：
> US-WORLD-001: As a 学生，I want 看到一个可视化的冒险地图，So that 我知道自己在哪儿、去过哪儿、接下来可以去哪儿。

**验收标准**：
- AC-WORLD-001-1: Given 学生登录进入主页，When 页面加载完成，Then 展示当前世界的冒险地图，已解锁关卡高亮、当前关卡闪烁、锁定关卡灰色。
- AC-WORLD-001-2: Given 学生刚完成一关，When 返回地图，Then 新解锁的关卡节点播放"点亮"动画（神奇树屋式钩子），持续约 3 秒。
- AC-WORLD-001-3: Given 学生累计完成一个世界（≥10 关），When 最后一关完成，Then 新世界在地图上"浮现"（世界进化机制），赠送新载具皮肤。

#### REQ-WORLD-002：六段式关卡叙事模板

**优先级**：Must have

**用户故事**：
> US-WORLD-002: As a 学生，I want 每关的冒险都有一个完整的故事（入口→到达→冲突→解题→胜利→钩子），So that 我不是在"做题"而是在"经历一场冒险"。

**验收标准**：
- AC-WORLD-002-1 (入口段): Given 学生点击进入关卡，When 关卡开始，Then 展示传送门/载具启动动画，博士说出引导语（含当日地点和悬念），时长 ≤ 30 秒。
- AC-WORLD-002-2 (到达段): Given 入口段结束，When 场景展开，Then 载具变形着陆，展示新场景的视觉+听觉+触觉描写（≥4 句感官描述）。
- AC-WORLD-002-3 (冲突段): Given 场景展开完毕，When 冲突呈现，Then 以 NPC 对话方式引出"数学案件"（至少 1 个拟人化数学概念 NPC 登场），学生明确知道"我要解决什么问题"。
- AC-WORLD-002-4 (解题段): Given 冲突已明确，When 学生开始解题，Then **文学线（剧情推进）和数学线（分步引导解题）交替出现**，每关 3-5 道题，每道题对应一个剧情节点。
- AC-WORLD-002-5 (胜利段): Given 所有题目完成，When 最后一题通过，Then 剧情展示冲突解决（如"分数王国内战平息"），博士情感肯定 + 知识总结，获得奖励动画（印章/装备碎片/新角色），时长 ≤ 45 秒。
- AC-WORLD-002-6 (钩子段): Given 胜利段结束，When 展示奖励后，Then 新线索出现（地图新区域亮起 / 新求救信号 / 神秘物品），悬念尾句，明日预告，自然引导"明天再来"。

#### REQ-WORLD-003：三种闯关模式

**优先级**：Must have

**用户故事**：
> US-WORLD-003: As a 学生，I want 不同风格的冒险模式轮换出现，So that 我不会觉得无聊，每次都有新鲜感。

**验收标准**：
- AC-WORLD-003-1 (英雄模式): Given 进入英雄模式关卡，When 冲突呈现，Then 数学世界出现危机，学生是"拯救者"（李毓佩式侦探/英雄叙事），解题 = 拯救世界。
- AC-WORLD-003-2 (导师模式): Given 进入导师模式关卡，When 冲突呈现，Then 出现一个遇到困难的小 NPC 角色，学生需要"教会"它数学知识（Teaching effect，学习收益 1.56×），解题 = 帮助同伴。
- AC-WORLD-003-3 (探索模式): Given 进入探索模式关卡，When 场景展开，Then 自由探索地图，发现隐藏的数学彩蛋（数学史故事/趣味谜题），非强制性答题，纯好奇心驱动。
- AC-WORLD-003-4 (轮换比例): 三种模式按 3:1:1 比例自动轮换，每完成一个大地图（≥10 关）强制切换模式。

#### REQ-WORLD-004：世界观四阶段进化

**优先级**：Should have（v1 实现阶段 1-2，v2 实现阶段 3-4）

**用户故事**：
> US-WORLD-004: As a 学生，I want 冒险世界不断扩展和变化，So that 我不会"通关"后无事可做，总有新世界等着探索。

**验收标准**：
- AC-WORLD-004-1 (阶段1-新手村): Given 新用户，When 入学，Then 从"魔法森林"新手村开始（入门引导、基础算术）。
- AC-WORLD-004-2 (阶段2-世界扩展): Given 完成新手村全部关卡，When 返回地图，Then 新世界解锁：海底王国（几何）、星际空间站（代数）、古文明遗迹（测量与比例）。
- AC-WORLD-004-3 (载具变形): 每个世界使用不同的冒险载具（魔法森林→扫帚，海底→潜水器，星际→飞船），载具变形动画 ≤ 5 秒。

### 4.3 模块三：三层解耦架构

#### REQ-ENGINE-001：决策引擎（Layer 1 — 零 LLM 成本）

**优先级**：Must have

**用户故事**：
> US-ENGINE-001: As a 系统，I want 知识点选择、难度调节、对错判定全部由算法驱动，So that 高频决策零 LLM 成本且可解释。

**验收标准**：
- AC-ENGINE-001-1 (知识点选择): Given 需要生成新关卡，When 决策引擎运行，Then 基于掌握度矩阵 + 遗忘曲线 + 进度状态机选择目标知识点，算法运算时间 < 50ms。
- AC-ENGINE-001-2 (难度调节): Given 选定知识点，When 计算目标难度，Then 使用 IRT 模型计算目标难度 θ，使 P(正确) ∈ [0.50, 0.70]（甜蜜点区间）。
- AC-ENGINE-001-3 (对错判定): Given 学生提交答案，When 规则引擎判定，Then 对数学答案进行确定性验证（数值等价性检查、单位检查、精度容差），判定时间 < 10ms。
- AC-ENGINE-001-4 (情绪保护): Given 学生连续答错 2 题，When 下一题生成，Then 自动降低一个难度档位，题目附带更多视觉提示（Zearn 模式：给台阶不打回原形）。

#### REQ-ENGINE-002：RAG 约束层（Layer 2 — 结构化约束包）

**优先级**：Must have

**用户故事**：
> US-ENGINE-002: As a 系统，I want 双向量库检索 + 知识图谱查询组装为结构化的"关卡约束包"，So that LLM 生成的内容不超纲、不偏离、不重复。

**验收标准**：
- AC-ENGINE-002-1 (公共库检索): Given 目标知识点 ID，When RAG 检索，Then 从公共库返回：课标要求、3 个典型例题、常见易错点、前置依赖知识点。
- AC-ENGINE-002-2 (私有库检索): Given 当前用户 ID，When RAG 检索，Then 从私有库返回：该知识点历史答题情况、上次错误类型、已生成剧情标签（排重用）、用户偏好。
- AC-ENGINE-002-3 (KG 查询): Given 目标知识点，When 查询知识图谱，Then 返回前置依赖列表（确保学生已掌握）、后续扩展列表（为下一关铺垫）、关联易错点。
- AC-ENGINE-002-4 (约束包组装): Given 三层检索结果，When 组装约束包，Then 输出结构化 JSON（含：知识点 ID、难度参数、禁止标签列表、角色配置、输出模板），组装时间 < 200ms。

#### REQ-ENGINE-003：LLM 生成层（Layer 3 — 仅负责内容创作）

**优先级**：Must have

**用户故事**：
> US-ENGINE-003: As a 系统，I want LLM 只接收"关卡约束包"并按六段式模板生成叙事内容，So that LLM 不被要求做它不擅长的精确决策。

**验收标准**：
- AC-ENGINE-003-1 (Prompt 结构): 每次 LLM 调用使用固定 System Prompt（数学博士人设 + 六段式叙事模板 + 禁用词清单）+ 关卡约束包 JSON。
- AC-ENGINE-003-2 (后置校验): Given LLM 生成完毕，When 后置校验运行，Then 检查：(a) 生成内容知识点与约束包匹配度 ≥ 90%，(b) 无超纲内容（KG 范围检查），(c) 无禁用词（"请计算""你错了""正确答案是"等），任一不通过触发重新生成。
- AC-ENGINE-003-3 (输出格式): LLM 输出结构化 JSON，包含 6 个 Section（对应六段式），每个 Section 含剧情文本数组、数学题数组（含分步提示）、角色对话数组。

#### REQ-ENGINE-004：模型分级调用与成本控制

**优先级**：Must have

| 调用场景 | 频率 | 模型 | 延迟要求 |
|----------|------|------|----------|
| 关卡剧情生成 | 低频（~1 次/天/用户） | deepseek-v4-pro | < 30s |
| 博士对话引导 | 中频（每次交互） | deepseek-v4-flash | < 3s |
| 课程大纲/世界观初始化 | 超低频（1 次/学期） | deepseek-v4-pro | 无实时要求 |
| 对错判定 | 高频（每次答题） | **规则引擎（零 LLM）** | < 10ms |
| 知识点选择 | 高频（每关） | **算法计算（零 LLM）** | < 50ms |
| 排重检查 | 高频（每关） | **哈希索引（零 LLM）** | < 10ms |
| Embedding | 低频（内容入库时） | DeepSeek Embedding API | < 5s |

**验收标准**：
- AC-ENGINE-004-1: LLM 成本相比"全链路 LLM 方案"降低 ≥ 60%，通过 prompt caching 使缓存命中率 > 80%。

### 🔴 需要 PO 确认（选择题）

**决策 #4：前端技术栈**

> 影响：开发速度、团队招聘、跨平台能力、动画表现力

| 选项 | 方案 | 优势 | 劣势 |
|------|------|------|------|
| **A (推荐)** | **React + TypeScript** | 生态最丰富、招聘容易、Three.js/GSAP 动画库成熟、跨平台可选 React Native | 包体积较大 |
| B | Vue 3 + TypeScript | 上手简单、中文社区强、性能好 | 游戏化动画库不如 React 生态丰富 |
| C | Flutter | 真跨平台（iOS/Android/Web）、动画性能最佳 | Web 端体验不如原生 Web 框架、学习曲线陡 |

**推荐理由**：本项目需要丰富的动画和游戏化交互（地图、载具变形、角色动画），React 生态的 Framer Motion + Three.js + PixiJS 是成熟选择。Vue 也可行但动画生态略弱。Flutter 跨平台优势大但 v1 聚焦 Web 的情况下引入复杂度过高。

**决策 #6：记忆模块存储方案**

> 影响：读写性能、数据模型灵活性、运维复杂度

| 选项 | 方案 | 优势 | 劣势 |
|------|------|------|------|
| **A (推荐)** | **Redis 热存储 + PostgreSQL 冷存储** | 高频读写毫秒级、iPET 三层记忆需要不同策略、自然支持 TTL | 多一层基础设施 |
| B | PostgreSQL JSONB 统一 | 架构简单，单库管理 | 高频读写性能不如 Redis，JSONB 深层查询效率低 |

**推荐理由**：记忆模块的核心场景是"每次对话都要读写用户近期记忆"，这是典型的热数据场景。iPET 论文也验证了分层记忆的必要性。PostgreSQL JSONB 作为冷存储做持久化，Redis 做热缓存，各司其职。
