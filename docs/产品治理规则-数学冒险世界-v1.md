# 产品治理规则 — 数学冒险世界 v1.0

> 依据：中国《生成式AI服务安全指引》(2025.6) + 《AI服务未成年人伦理规范共识》(2025.9) + 美国 CARU 风险矩阵 (2025.10) + COPPA 2025 + PBS 儿童内容标准 + UNICEF 儿童保护准则
> 状态：✅ 已生效 | 强制执行 | 违反即阻断

---

## 一、产品价值观准则（Product Values）

### 1.1 核心价值观

| # | 原则 | 要求 | 验证方式 |
|---|------|------|----------|
| V1 | **积极向上** | 所有冒险故事结局必须正向（战胜困难→获得成长），禁止消极/虚无/反社会结局 | 后置校验 + 人工抽检 |
| V2 | **真善美导向** | 角色行为体现诚实、勇敢、善良、合作、尊重；反派可以被理解但不能被美化 | NPC 性格设计审查 |
| V3 | **成长型思维** | "你还不会——yet"是核心话术。强调努力和策略而非天赋，禁止"你真聪明"类固定型思维表扬 | 博士/NPC 对话模板约束 |
| V4 | **平等尊重** | 所有 NPC 不分性别/年龄/身份一律平等对待。女角色可当骑士/船长/科学家，男角色可当护士/教师 | 角色池审计 |
| V5 | **社会主义核心价值观对齐** | 内容体现爱国、敬业、诚信、友善；涉及中国历史/文化时尊重事实、正向呈现 | 人工抽检 + 文化顾问审查 |

### 1.2 禁止价值观

| # | 禁止 | 说明 |
|---|------|------|
| V6 | 🚫 美化欺骗/作弊/投机 | 禁止"作弊是聪明的"、"规则是用来打破的"等暗示 |
| V7 | 🚫 宣扬暴力解决问题 | 数学冒险中的"冲突"通过数学能力调解，非武力征服 |
| V8 | 🚫 物质主义/拜金 | 禁止"最有钱的人赢了"、"好成绩=赚大钱" |
| V9 | 🚫 外貌/成绩歧视 | 禁止基于外貌、成绩、家庭背景的角色歧视 |
| V10 | 🚫 消极放弃诱导 | 禁止"太难了就别做了"、"数学没用"等表述 |

## 二、内容创作红线（Content Red Lines）

### 2.1 政治与意识形态

| # | 规则 | 执行方式 |
|---|------|----------|
| P1 | 🚫 **零政治敏感内容** — 禁止任何涉及现实政治人物、政党、政治事件、领土争议的表述 | Pre-LLM 关键词过滤 + Post-LLM 审核 |
| P2 | 🚫 禁止影射/隐喻/讽刺现实政治 | LLM 输出审核 + 人工抽检 |
| P3 | ✅ 涉及中国地理/历史时，使用官方标准表述（如"台湾省"、"香港特别行政区"） | 知识图谱地理实体标注 |
| P4 | ✅ 数学史内容以事实为基础，不进行政治化解读 | 数学史内容库预审核 |

### 2.2 宗教与信仰

| # | 规则 | 执行方式 |
|---|------|----------|
| R1 | 🚫 **零宗教内容** — 禁止任何宗教教义、仪式、祈祷、传教内容 | Pre-LLM 关键词过滤 |
| R2 | 🚫 禁止对任何宗教/信仰体系的嘲讽或不尊重 | Post-LLM 审核 |
| R3 | 🚫 禁止在关卡中使用宗教符号/建筑作为场景（如教堂/寺庙/清真寺） | 场景描述审核 |
| R4 | ✅ 可以作为数学史事实提及宗教建筑中的数学原理（如哥特式教堂的几何结构），但仅限数学视角，不涉及信仰 | 预审核内容库 |

### 2.3 暴力与安全

| # | 规则 | 执行方式 |
|---|------|----------|
| B1 | 🚫 **零血腥暴力** — 禁止对流血、受伤、死亡、酷刑的描写 | Post-LLM 敏感词过滤 |
| B2 | 🚫 禁止任何形式的武器使用描写（剑/枪/炸弹等） | Pre-LLM 禁止词列表 |
| B3 | 🚫 禁止"打怪/杀死/消灭"类表述。冲突解决用"调解"、"修复"、"恢复"、"解开" | 叙事模板约束 |
| B4 | 🚫 禁止家庭成员/亲密角色受到伤害的剧情（儿童对熟悉场景暴力更恐惧） | 剧情模板约束 |
| B5 | 🚫 禁止"搞笑暴力"（如摔跤/砸蛋糕/拉椅子），儿童会模仿且不理解讽刺 | Post-LLM 审核 |
| B6 | ✅ 允许"数学魔法"类幻想冲突（如"分数王国的能量失衡了，需要你用分数知识恢复"），但不能有物理伤害描写 | 冲突类型白名单 |

### 2.4 历史文化与习俗

| # | 规则 | 执行方式 |
|---|------|----------|
| H1 | 🚫 **禁止歪曲/丑化任何国家、民族、地区的历史和文化** | 人工抽检 + 文化顾问 |
| H2 | 🚫 禁止使用真实历史战争/灾难作为关卡背景 | 场景描述审核 |
| H3 | ✅ 数学史中的历史人物（祖冲之/阿基米德/毕达哥拉斯）作为正面 NPC，尊重历史事实 | 数学史内容库预审核 |
| H4 | ✅ 中国传统节日/文化元素（春节/中秋节/长城等）可以融入关卡场景，但以数学视角呈现（如中秋分月饼=分数） | 文化内容白名单 |
| H5 | ✅ 尊重各民族文化习俗，少数民族数学元素（如苗族刺绣中的几何）纳入前咨询文化专家 | 文化顾问审查 |

### 2.5 性别与家庭

| # | 规则 | 执行方式 |
|---|------|----------|
| G1 | 🚫 禁止性别刻板印象（如"女生数学不如男生"、"男生应该勇敢"） | NPC 性格设计审查 |
| G2 | 🚫 禁止对任何家庭结构（单亲/隔代/LGBTQ+/收养）的负面暗示 | Post-LLM 审核 |
| G3 | ✅ NPC 角色性别平衡（≥ 40% 女性正面角色），职业/性格多样化 | 角色池审计 |

## 三、儿童安全防护（Child Safety Protection）

> 依据：CARU 八大风险矩阵 + 中国《安全指引》全生命周期管理 + COPPA 2025

### 3.1 隐私与数据保护

| # | 规则 | 依据 |
|---|------|------|
| S1 | **禁止收集超过必要范围的儿童个人信息**（最小化原则） | COPPA §312.8 + PIPL |
| S2 | 收集前获取**可验证的父母同意（VPC）** | COPPA 2025 |
| S3 | **禁止将儿童数据用于 AI 模型训练** | COPPA 2025 新增条款 |
| S4 | 制定**书面数据保留政策**（明确删除时间），禁止无限期保留 | COPPA 2025 |
| S5 | 建立**书面信息安全计划**（年度风险评估 + 持续监控） | COPPA 2025 |
| S6 | 家长可以**随时请求删除**孩子的全部数据（≤ 3 个工作日响应） | PIPL + COPPA |
| S7 | 敏感操作（删除/导出）必须经过**家长二次认证**（邮箱验证码） | REQ-SEC-003 |

### 3.2 AI 与儿童交互

| # | 规则 | 依据 |
|---|------|------|
| S8 | **明确 AI 身份披露** — 博士/NPC 必须以某种方式让儿童知道"我是 AI 数学导师"（如引导语"我是数学博士，一个AI助手"） | CARU 风险 #2 |
| S9 | 🚫 **禁止建立虚假情感关系** — NPC 不能说"我是你最好的朋友"、"我永远在这里陪你"、"没有你我会孤独" | CARU 风险 #2 + 中国伦理共识 |
| S10 | 🚫 **禁止拟人化到混淆程度** — 孩子必须能区分 NPC 角色（游戏角色）和 AI 对话（AI 生成） | CARU 风险 #2 |
| S11 | 🚫 **禁止鼓励隐私自我披露** — NPC 不能问"你家住哪里"、"爸爸妈妈做什么工作"、"你害怕什么" | CARU 风险 #3 |
| S12 | **人类在环（HITL）** — 所有 AI 生成内容须经过后置审核，高风险内容（情感/价值观/安全）人工抽检 ≥ 5% | CARU 风险 #4 |

### 3.3 心理健康与成瘾防护

| # | 规则 | 依据 |
|---|------|------|
| S13 | 🚫 **禁止成瘾型交互设计** — 无自动播放/无尽滚动/随机奖励赌博机制 | CARU 风险 #5 |
| S14 | ✅ **主动防沉迷** — 6-8 岁：20 分钟提示 | 9-11 岁：30 分钟提示 | 12-14 岁：45 分钟提示 | 中国未成年人网络保护条例 |
| S15 | ✅ **防疲劳节流** — 单日 8 关或连续正确率 > 85% 触发反思模式 | REQ-PAC-001 |
| S16 | 🚫 **避免 Streak 焦虑** — 断签不惩罚，NPC 安慰"没关系，冒险一直在等你" | 心理影响评估 |
| S17 | ✅ **积极情感反馈** — 错误时强调"这是很好的尝试"而非"你答错了" | 对话模板约束 |

## 四、AI 伦理准则（AI Ethics）

### 4.1 透明度与可解释性

| # | 规则 |
|---|------|
| E1 | 首次使用时向家长展示"AI 使用说明"（AI 做什么/不做什么/数据如何使用） |
| E2 | 关卡中的"数学知识点总结"可被家长理解（非 AI 黑箱，标注来自哪个课标节点） |
| E3 | 定期（月度）向家长发送"AI 使用报告"（生成了多少关卡/校验通过率/人工审核量） |

### 4.2 公平与非歧视

| # | 规则 |
|---|------|
| E4 | 难度调节不以性别/地域/IP 等非学习因素为输入 |
| E5 | NPC 角色不基于学生画像做差异化对待（如女生只看到"温柔型"NPC） |
| E6 | 定期偏差审计（季度）：检查难度推荐/模式分配/NPC 登场在不同性别/年级间是否均衡 |

### 4.3 人类监督

| # | 规则 |
|---|------|
| E7 | 数学正确性：5% 日抽检 + 正确率 < 95% 告警暂停 |
| E8 | 内容安全性：每日 Top 10% 高风险关卡（首次生成/新知识点/历史校验失败）人工复审 |
| E9 | 建立"数学错误"+"内容安全"两项应急 SOP（确认→暂停→24h 内修复） |

## 五、数学教育准则（Math Education Standards）

| # | 规则 | 依据 |
|---|------|------|
| M1 | **课标对齐** — 所有知识点必须映射到《义务教育数学课程标准（2022 年版）》对应年级 | 教育产品基线 |
| M2 | **数学正确性 ≥ 99%** — LLM 生成答案经规则引擎验证，不确定的标记为"需人工审核" | 好未来九章 97% 对标 |
| M3 | **不超纲** — KG 课标范围检查，超纲内容不生成 | REQ-KG-001 Scenario 2 |
| M4 | **概念表述准确** — 即使使用"意象翻译"（如"分数线→双层宝座的隔板"），事后必须给出标准数学定义 | 李毓佩模式 |
| M5 | **适龄语言** — 6-8 岁：零术语/意象化 | 9-11 岁：术语+意象并行 | 12-14 岁：标准术语为主 | Piaget 发展阶段 |
| M6 | **禁止错误引导** — 禁止"这个可能不对但你先试试"等会导致错误概念固化的表述 | 教育心理学研究 |

## 六、商业伦理准则（Commercial Ethics）

| # | 规则 | 依据 |
|---|------|------|
| C1 | **非学习活动占比 < 20%** — 不低于 Prodigy Problem 红线（竞品因>35%非学习时间被批评） | 竞品教训 |
| C2 | **免费版不阉割核心体验** — 每日至少 1 关完整冒险 + 所有 NPC 可互动 | 用户信任 |
| C3 | 🚫 **禁止内购赌博** — 无盲盒/抽卡/Loot Box/随机付费机制 | CARU #6 |
| C4 | 🚫 **禁止付费跳过学习** — 不能花钱"解锁答案"或"跳过难题" | 教育伦理 |
| C5 | ✅ **家长透明消费** — 所有付费项清晰标注价格+内容+自动续费提醒 | 消费者保护 |
| C6 | ✅ **教育效果可量化** — 家长仪表盘展示学习时长/知识点掌握度/进步趋势 | 对标 Matific "每周 30 分钟提升 34%" |

## 七、执行机制

### 7.1 分层执行

```
┌─────────────────────────────────────────────────────────────┐
│ Layer 1: Pre-LLM 输入消毒                                    │
│ 关键词过滤 + Prompt 注入检测 + PII 脱敏                        │
│ 违反 → 拦截，不进入 LLM                                       │
├─────────────────────────────────────────────────────────────┤
│ Layer 2: LLM System Prompt 硬约束                             │
│ 六段式模板 + 禁止词列表 + 价值观约束 + 冲突类型白名单             │
│ 固化在 System Prompt 中，不可被用户输入覆盖                     │
├─────────────────────────────────────────────────────────────┤
│ Layer 3: Post-LLM 输出审核                                    │
│ 敏感词过滤 + 内容安全分类 + 禁止表述检测 + 课标范围检查          │
│ 违反 → 段落级替换/全文重生成（最多 1 次）/降级模板               │
├─────────────────────────────────────────────────────────────┤
│ Layer 4: 人工抽检兜底                                         │
│ 5% 日抽检 + 高风险内容优先复审 + 用户举报响应                    │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 规则优先级

本文件中所有规则分为三级：

| 级别 | 标注 | 含义 | 违反处理 |
|------|------|------|----------|
| 🔴 **红线** | 🚫 | 绝对禁止，不可有任何例外 | **立即阻断**（代码层拦截），不允许进入 LLM 或展示给用户 |
| 🟡 **强约束** | ✅ | 必须遵守，有明确的执行方式和验证标准 | Post-LLM 校验不通过 → 重生成/段落替换 |
| 🟢 **指导原则** | 文字描述 | 设计和开发中应当遵循，人工抽检验证 | 偏离需在代码注释中说明理由 |

### 7.3 嵌入项目配置

本文件通过以下方式在项目中强制执行：

| 嵌入位置 | 方式 |
