## 1. 引言

### 1.1 目的

本文档旨在明确"家庭纸质图书管理系统"的软件需求，为系统设计、开发、测试及验收提供完整依据。系统面向家庭或小型私人藏书场景，帮助用户高效管理纸质图书的入库、检索、借阅与归还等日常操作。本文档遵循 IEEE Std 830-1998 标准结构编写，作为项目团队（开发、测试、运维）与利益相关方之间的需求共识基线。

### 1.2 文档约定

- **需求编号规则**：功能需求以 `FR-` 为前缀，非功能需求以 `NFR-` 为前缀，后接三位数字序号（如 FR-001）。
- **优先级定义**：P0（必须实现，系统核心功能）、P1（重要功能，建议首发版本实现）、P2（次要功能，后续迭代实现）。
- **用例编号规则**：以 `UC-` 为前缀，后接两位数字序号（如 UC-01）。
- **假设与约束编号**：假设以 `ASM-` 为前缀，约束以 `CST-` 为前缀。
- **关键词约定**：本文档中"必须（shall）"、"应该（should）"、"可以（may）"的使用遵循 RFC 2119 规范。

### 1.4 产品范围

本系统覆盖以下核心功能范围：

- 用户权限管理（管理员/普通家庭成员）
- 图书的增、删、改、查（CRUD）
- 基于 ISBN 的智能扫码入库（调用国产大模型 + 国图/豆瓣 API）
- 简单借阅登记（借书、还书、借阅记录查询）
- 多品类图书支持（单本、套装、丛书、期刊、报纸、连环画、工具书等）

**系统不包含**：电子书管理、在线阅读、社交分享、图书漂流、付费借阅、物流配送等。

## 2. 综合描述

### 2.1 产品前景

家庭纸质图书管理系统旨在解决家庭或小型私人藏书场景中图书管理混乱、借阅无记录、查找困难等痛点。随着家庭藏书量的增长（数百至数千册），传统的手工登记方式已无法满足高效管理需求。本产品通过 ISBN 扫码智能入库、多品类图书管理、借阅登记与追踪等功能，帮助家庭用户实现图书资产的数字化管理，提升图书查找效率，规范家庭内部借阅行为。

本产品定位为轻量级、易部署的家庭级应用，运行于家庭局域网或小型服务器环境，不依赖云原生架构，适合技术背景有限的家庭用户使用。

### 2.2 功能概述

系统按业务领域划分为以下六大功能模块：

| 模块 | 核心功能 | 优先级 |
|------|----------|--------|
| 用户管理 | 管理员/普通用户双角色、用户 CRUD、登录认证（JWT Token）、账号启停 | P0 |
| 图书管理 | 手动录入、ISBN 扫码智能入库、图书信息编辑/逻辑删除、多条件组合检索、多品类管理（单本/套装/丛书/期刊/报纸/连环画/工具书）、物理位置标记 | P0 |
| 借阅管理 | 借书登记、还书登记、在借列表查看、借阅历史查询、逾期标记与催还 | P0 |
| 数据统计 | 总藏书量、各品类数量、在借数量等概览数据、图书清单导出（Excel/CSV） | P1 |
| 系统管理 | 操作日志查看、数据备份与恢复 | P2 |
| 基础设施 | 认证鉴权、异常处理、配置管理、外部 API 客户端封装 | P0 |

### 2.3 用户特征

| 用户角色 | 描述 | 技能水平 | 使用频率 |
|----------|------|----------|----------|
| 管理员 | 家庭图书管理员，通常为家庭中负责图书管理的成员 | 中等计算机操作能力 | 每日/每周 |
| 普通用户 | 家庭成员，可查询图书、查看个人借阅记录 | 基本计算机操作能力 | 每周/每月 |

典型用户规模：家庭场景下，用户数通常为 2~10 人，图书总量数千本。

|----------|------|
| 部署模式 | 前后端分离部署，后端提供 RESTful API，前端为 SPA 应用 |
| 服务器 | 家庭局域网内小型服务器（如 NAS、树莓派、旧 PC），或单机部署 |

| 网络环境 | 家庭局域网，需互联网访问以调用国产大模型 API 和国图/豆瓣数据源 |

### 2.5 设计与实现约束

| 编号 | 约束内容 |
|------|----------|
| CST-001 | 必须使用国产在线大模型（如通义千问、文心一言、智谱 GLM 等），不得使用 OpenAI 等境外 AI 服务 |
| CST-002 | 数据库必须使用 MySQL 8.0+，不得使用 NoSQL 数据库 |
| CST-003 | 前端必须使用 Vue 3 框架 |
| CST-004 | 后端必须使用 Python 语言（推荐 FastAPI 框架） |
| CST-005 | 系统需在家庭局域网或小型服务器上部署，不要求云原生架构 |
| CST-006 | 前后端通信必须使用 RESTful API，数据格式为 JSON |
| CST-007 | 用户密码必须加密存储（推荐 bcrypt 或 Argon2） |
| CST-008 | 所有 API 接口需鉴权（登录接口除外），使用 JWT Bearer Token |
| **CST-009** | **系统必须实现基于角色的访问控制（RBAC）：管理员专属接口（如用户管理、系统管理）必须验证角色，普通用户仅能访问公开及个人数据。** |

### 2.6 假设与依赖

| 编号 | 假设/依赖内容 |
|------|---------------|
| ASM-001 | 用户拥有可正常访问互联网的环境，用于调用国产大模型 API 和国图/豆瓣数据源 |
| ASM-002 | 国产大模型 API 能够稳定返回结构化的图书元数据（JSON 格式） |
| ASM-003 | 国图/豆瓣接口对个人或小规模应用无严格调用频率限制（或可通过大模型代理规避） |
| ASM-004 | 家庭场景下，图书总量数千本，用户数十人，系统无需高并发设计 |
| ASM-005 | 扫码功能仅在手机端使用，PC 端可提供手动输入 ISBN 的替代方案 |
| ASM-006 | 图书封面图片从数据源获取后以 URL 形式存储，不本地保存图片文件 |
| ASM-007 | 期刊/报纸使用 ISSN 作为标识，若无 ISSN 则使用"刊名+年份+期号"作为唯一标识 |
| ASM-008 | 连环画/小人书若无 ISBN，允许管理员手动录入基本信息 |
| ASM-009 | 国产大模型 API 密钥需安全存储，不暴露在前端代码中 |

### 3.2 硬件接口

| 需求编号 | 需求描述 |
|----------|----------|
| HI-001 | 系统需通过前端浏览器调用设备摄像头，用于 ISBN 条码扫描 |
| HI-002 | 扫码功能使用前端摄像头 API（MediaDevices.getUserMedia）+ 条形码识别库（QuaggaJS 或 ZXing） |
| HI-003 | 系统不依赖专用扫码硬件，仅使用手机/PC 自带摄像头 |

### 3.3 软件接口

**3.3.1 接口设计原则**

- 所有接口以 `/api/v1` 为前缀，使用 JSON 作为请求与响应体格式。
- 统一采用 `application/json` Content-Type，字符编码 UTF-8。
- 鉴权通过 HTTP Header `Authorization: Bearer <token>` 传递，登录接口除外。
- 分页查询使用 `page`（页码，从 1 开始）和 `size`（每页条数，默认 20）参数，返回统一分页结构。

**3.3.2 接口列表**

| 需求编号 | 方法 | 路径 | 描述 | 鉴权要求 |
|----------|------|------|------|----------|
| SI-001 | POST | /api/v1/auth/login | 用户登录，返回 JWT | 否 |
| SI-002 | GET  | /api/v1/auth/me | 获取当前登录用户信息 | 是 |
| SI-003 | POST | /api/v1/books/scan | 接收 ISBN，触发 AI 元数据获取 | 管理员 |
| SI-004 | POST | /api/v1/books | 手动创建图书（含多品类） | 管理员 |
| SI-005 | GET  | /api/v1/books/{id} | 查询单本图书详情 | 是 |
| SI-006 | PUT  | /api/v1/books/{id} | 更新图书信息 | 管理员 |
| SI-007 | DELETE | /api/v1/books/{id} | 逻辑删除图书 | 管理员 |
| SI-008 | GET  | /api/v1/books | 多条件检索图书（分页） | 是 |
| SI-009 | POST | /api/v1/borrows | 借书登记 | 管理员 |
| SI-010 | PUT  | /api/v1/borrows/{id}/return | 还书登记 | 管理员 |
| SI-011 | GET  | /api/v1/borrows | 查询借阅记录（在借/历史） | 见说明 |
| SI-012 | GET  | /api/v1/users | 查询用户列表 | 管理员 |
| SI-013 | POST | /api/v1/users | 创建新用户 | 管理员 |
| SI-014 | PUT  | /api/v1/users/{id} | 更新用户信息（含密码重置） | 管理员/用户本人 |
| SI-015 | PUT  | /api/v1/users/{id}/status | 启用/禁用用户 | 管理员 |
| SI-016 | GET  | /api/v1/statistics | 获取藏书统计概览 | 是 |
| SI-017 | GET  | /api/v1/logs | 查看操作日志 | 管理员 |
| SI-018 | POST | /api/v1/system/backup | 启动数据库备份 | 管理员 |
| SI-019 | POST | /api/v1/system/restore | 从备份文件恢复 | 管理员 |

**说明**：
- SI-011（借阅记录查询）：管理员可查看所有记录，普通用户仅可查看本人的借阅记录，通过请求中的 `user_id` 查询参数（普通用户忽略该参数并强制使用当前用户ID）或由后端自行过滤。
- SI-014（用户更新）：用户可修改自己的基本信息（昵称、密码等），不可修改角色；管理员可修改任意用户信息并重置密码。
- 所有"是"鉴权接口均需验证 JWT 有效性和权限。

**3.3.3 请求/响应格式示例**

**登录 (SI-001)**  
请求体：
```json
{
  "username": "string",
  "password": "string"
}
```
响应（成功）：
```json
{
  "code": 0,
  "message": "登录成功",
  "data": {
    "access_token": "eyJhbGciOiJI...",
    "token_type": "bearer",
    "user_info": {
      "id": 1,
      "username": "admin",
      "nickname": "管理员",
      "role": "admin"
    }
  }
}
```

**图书检索 (SI-008)**  
请求参数（query string）：
- `keyword`（可选，模糊匹配书名、作者、ISBN）
- `category`（可选，品类）
- `location`（可选，物理位置）
- `status`（可选，在库/已借出/全部）
- `page`（默认 1），`size`（默认 20）

响应：
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "total": 150,
    "page": 1,
    "size": 20,
    "items": [
      {
        "id": 1,
        "title": "示例书名",
        "author": "作者",
        "isbn": "978-7-xxx-xxxxx-x",
        "cover_url": "https://...",
        "status": "available",
        "location": "A-01-03"
      }
    ]
  }
}
```

**借书登记 (SI-009)**  
请求体：
```json
{
  "user_id": 2,
  "book_id": 10,
  "borrow_date": "2025-07-15"  // 可选，默认当天，管理员可补录
}
```
响应（成功）返回借阅记录基本信息，包含自动计算的应还日期。

**还书登记 (SI-010)**  
路径参数 `id` 为借阅记录 ID。  
请求体：
```json
{
  "return_date": "2025-07-20"   // 可选，默认当天
}
```

**通用错误响应格式**  
```json
{
  "code": 10001,
  "message": "错误描述",
  "detail": {}  // 可选
}
```

### 3.4 错误码定义

| 错误码范围 | 类别 | 典型错误 |
|------------|------|----------|
| 0 | 成功 | — |
| 10001-10099 | 认证鉴权 | Token 无效、过期、权限不足 |
| 20001-20099 | 用户业务 | 用户名已存在、密码错误、账号锁定等 |
| 30001-30099 | 图书业务 | 图书不存在、ISBN 重复、状态异常等 |
| 40001-40099 | 借阅业务 | 图书不在库、无借阅记录、已还书等 |
| 50001-50099 | 外部服务 | 大模型调用失败、数据源超时等 |
| 60001-60099 | 系统级 | 数据库错误、文件操作失败等 |

具体错误码及含义在设计阶段细化。

### 3.5 通信接口

| 需求编号 | 需求描述 |
|----------|----------|
| CI-001 | 前端与后端通信使用 HTTP/HTTPS 协议（生产环境必须使用 HTTPS） |
| CI-002 | 所有 API 请求（除登录外）需在 HTTP Header 中携带 `Authorization: Bearer <token>` |
| CI-003 | 后端通过 HTTP 调用国产大模型 API（如通义千问、文心一言等），使用 API Key 鉴权 |

## 4. 系统特性

### 4.1 功能需求详情

#### 4.1.1 用户管理模块

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-001 | 系统支持管理员和普通用户两种角色 | P0 |
| FR-002 | 管理员可创建、编辑、禁用/启用普通用户账号 | P0 |
| FR-003 | 用户可修改个人密码和基本信息（昵称、头像） | P1 |
| FR-004 | 用户登录需验证用户名+密码，支持记住登录状态（JWT Token） | P0 |
| FR-005 | 连续 5 次登录失败后，账号临时锁定 15 分钟 | P1 |
| **FR-005a** | **管理员可为任何用户重置密码；重置后的密码需在首次登录时强制修改（可通过标记实现）** | P1 |

#### 4.1.2 图书管理模块

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| FR-006 | 支持手动录入图书信息（书名、作者、出版社、ISBN 等） | P0 |
| FR-007 | 支持通过手机摄像头扫描 ISBN 条码，自动识别 ISBN 号 | P0 |
| FR-008 | 扫描 ISBN 后，调用国产大模型智能解析并自动从国图/豆瓣获取图书元数据 | P0 |
| FR-009 | 支持对获取到的元数据进行人工确认和编辑后再入库 | P0 |
| FR-010 | 支持图书信息编辑（修改书名、作者、分类、位置等） | P0 |
| FR-011 | 支持删除图书（逻辑删除，保留操作记录） | P1 |
| FR-012 | 支持按书名、作者、ISBN、出版社、品类等条件组合检索 | P0 |
| FR-013 | 支持图书列表展示，含封面缩略图、基本信息 | P0 |
| FR-014 | 支持多品类图书的区分管理（单本/套装/丛书/期刊/报纸/连环画/工具书） | P1 |
| FR-015 | 套装书支持录入总册数和各分册信息 | P1 |
| FR-016 | 丛书支持关联展示同一丛书下的所有单本 | P2 |
