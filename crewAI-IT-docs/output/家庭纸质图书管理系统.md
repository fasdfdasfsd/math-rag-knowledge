## 1. 引言

### 1.1 目的

本文档是“家庭纸质图书管理系统”的软件需求规格说明书（Software Requirements Specification, SRS），遵循 IEEE Std 830-1998 标准编制。本文档旨在：

- 明确系统的功能需求与非功能需求；
- 定义系统与外部环境的接口；
- 为后续系统设计、编码实现、测试验证及验收交付提供统一依据；
- 确保项目团队与利益相关方对系统范围达成共识。

### 1.2 文档约定

- **需求编号规则**：功能需求以 `FR-` 为前缀，非功能需求以 `NFR-` 为前缀，后接三位数字序号（如 FR-001）。
- **优先级定义**：
  - **P0**：必须实现，系统核心功能，缺失则系统不可用；
  - **P1**：重要功能，建议在首版实现；
  - **P2**：次要功能，可在后续迭代中实现。
- **用例编号规则**：以 `UC-` 为前缀，后接三位数字序号（如 UC-001）。
- **关键词约定**：本文档中“必须（shall）”、“应当（should）”、“可以（may）”的使用遵循 RFC 2119 规范。
- **图表约定**：所有架构图、流程图使用 Mermaid 语法描述。

### 1.4 产品范围

本系统面向家庭场景，帮助家庭成员管理纸质藏书。系统覆盖以下业务范围：

- **用户权限管理**：家庭成员角色划分（管理员/普通成员），账号创建与登录；
- **图书全生命周期管理**：图书的增、删、改、查（CRUD），支持基于 ISBN 的智能录入（调用 AI 及第三方数据源）；
- **图书品类管理**：支持单本、套装、系列丛书、期刊、报纸、连环画、工具书等品类；
- **借阅登记与归还管理**：家庭成员内部借阅流程，无罚款机制；
- **数据统计**：藏书量、品类分布、借出状态等概览。

**系统不涉及**：电子书在线阅读、在线支付、社交分享、外部人员借阅等功能。

## 2. 综合描述

### 2.1 产品前景

家庭纸质图书管理系统旨在解决家庭藏书管理中的常见痛点：图书数量增长后难以查找、借出后忘记归还、重复购买、无法快速了解家庭藏书全貌等问题。系统通过 ISBN 扫描智能录入、分类管理、借阅追踪等功能，帮助家庭实现纸质图书的数字化管理，提升藏书利用效率。

系统定位为轻量级家庭应用，部署在家庭内网或轻量云服务器上，支持手机和 PC 浏览器访问，无需安装专用 App。

### 2.2 功能概述

系统按业务域划分为五大功能模块：

| 模块 | 核心功能 | 优先级 |
|------|----------|--------|
| 用户管理 | 账号注册/登录、角色权限管理、个人信息维护、密码管理 | P0 |
| 图书管理 | ISBN 扫描录入、AI 智能获取元数据、手动录入、编辑/删除、搜索/分页、品类管理、批量导入/导出 | P0/P1/P2 |
| 借阅管理 | 借阅申请、归还登记、状态检查、借阅历史查询、逾期提醒 | P0/P1/P2 |
| 数据统计 | 统计看板（总藏书量、品类分布、借出数量、逾期数量）、图表生成 | P1/P2 |
| 系统管理 | 操作日志记录与查询、系统健康检查、API 调用统计 | P1/P2 |

### 2.3 用户特征

| 用户角色 | 描述 | 技能水平 | 使用频率 | 权限范围 |
|----------|------|----------|----------|----------|
| 管理员 | 家庭中负责图书管理的成员（通常1~2人） | 具备基本计算机操作能力 | 每日或每周 | 全部权限，包括用户管理、图书录入/编辑/删除、借阅代操作 |
| 普通成员 | 家庭中其他成员（配偶、子女、长辈等） | 具备基本手机操作能力 | 按需使用 | 搜索图书、查看详情、发起借阅/归还、查看个人借阅记录 |

## 3. 外部接口需求

### 3.1 用户界面

#### 3.1.1 总体要求

- **UI-001**：系统界面必须支持响应式布局，自动适配手机（320px+）、平板（768px+）、PC（1024px+）屏幕尺寸。
- **UI-002**：系统必须提供中文界面，所有提示信息、错误消息、按钮标签等使用中文。
- **UI-003**：关键操作（借阅、归还、录入）必须提供明确的成功/失败反馈（如 Toast 提示、弹窗确认）。
- **UI-004**：系统界面风格应简洁清晰，符合家庭使用场景，避免过于复杂的交互。

#### 3.1.2 主要页面

| 页面 | 说明 | 访问角色 |
|------|------|----------|
| 登录页 | 用户名/手机号 + 密码输入，登录按钮，首次登录修改密码引导 | 所有用户 |
| 首页/看板 | 统计概览（总藏书量、借出数量、逾期数量）、快捷操作入口 | 所有用户 |
| 图书列表页 | 图书卡片/表格展示，支持搜索、分页、排序 | 所有用户 |
| 图书详情页 | 封面、元数据、借阅状态、借阅历史 | 所有用户 |
| 图书录入页 | ISBN 扫描界面 + 手动录入表单 | 管理员 |
| 借阅管理页 | 当前借出列表、历史借阅记录、借阅/归还操作 | 所有用户 |
| 用户管理页 | 家庭成员列表、创建/编辑/删除用户，密码重置入口 | 管理员 |
| 个人设置页 | 修改昵称、头像、密码 | 所有用户 |
| 操作日志页 | 系统操作日志查询 | 管理员 |

#### 3.1.3 ISBN 扫描界面

- **UI-005**：扫描界面必须调用手机摄像头，显示实时取景画面，并在画面中叠加扫描框。
- **UI-006**：扫描成功后，系统应发出提示音或震动反馈（如设备支持）。
- **UI-007**：扫描失败时，系统应提供“手动输入 ISBN”的备选入口。
- **UI-008**：扫描过程中应显示“正在查询图书信息…”的加载状态。

### 3.2 硬件接口

| 编号 | 接口描述 |
|------|----------|
| HI-001 | **摄像头**：系统通过 HTML5 的 `getUserMedia` API 调用设备摄像头，用于 ISBN 条形码扫描。无需专用扫码硬件。 |
| HI-002 | **存储设备**：系统运行于通用服务器硬件（家庭内网服务器或云服务器），无特殊硬件要求。封面图片等文件存储于本地磁盘或云对象存储。 |

### 3.3 软件接口

#### 3.3.1 第三方数据源接口

| 编号 | 接口名称 | 协议 | 数据格式 | 用途 | 降级策略 |
|------|----------|------|----------|------|----------|
| SI-001 | 中国国家图书馆 API | HTTPS / REST | JSON | 查询图书元数据（书名、作者、出版社等） | 超时或失败时切换至豆瓣 API；若豆瓣也失败，则标记“数据源不可用”，进入手动录入流程 |
| SI-002 | 豆瓣 API（非官方） | HTTPS / REST | JSON | 查询图书评分、简介、封面等 | 超时或失败时返回空数据，提示手动录入 |
| SI-003 | 国产大模型 API（文心一言/通义千问等） | HTTPS / REST | JSON | AI 智能体：ISBN 信息提取、OCR 识别、数据聚合 | 超时或失败时跳过 AI 处理，直接使用原始数据或空表单，由管理员确认/补全 |

#### 3.3.2 AI 智能体接口

- **SI-004**：后端通过 HTTP 调用国产大模型 API，将 ISBN 或用户输入信息发送至 AI 智能体。
- **SI-005**：AI 智能体负责依次查询国图 API 和豆瓣 API，聚合返回结构化图书元数据。实际调用由后端代理完成，AI 模型不直接访问外部 API，而是由后端提供数据并请求 AI 进行信息提取和结构化。
- **SI-006**：若国图/豆瓣均无数据，后端应将 ISBN 解析为有限的结构化信息（如国家代码、出版社代码）作为上下文传递给 AI，由 AI 尝试推理部分元数据；无论 AI 是否返回有效数据，系统都必须最终展示预填信息并提示管理员手动补全必填项。
- **SI-007**：AI 调用采用异步任务模式，避免阻塞用户操作。前端通过轮询或 WebSocket 获取处理进度。

#### 3.3.3 前后端接口

- **SI-008**：前后端通过 HTTPS / RESTful API 进行通信，数据格式为 JSON。
- **SI-009**：鉴权方式为 JWT Token，前端在请求头中携带 `Authorization: Bearer <token>`。
- **SI-010**：全局错误响应格式统一为：

```json
{
  "code": 400,
  "message": "具体错误信息",
  "detail": {}
}
```

- **SI-011**：主要 API 端点定义（首版必须实现）：

| 端点 | 方法 | 说明 | 鉴权要求 |
|------|------|------|----------|
| `/api/auth/login` | POST | 用户登录，返回 JWT Token 与用户信息 | 无需 |
| `/api/auth/refresh` | POST | 刷新即将过期的 Token（若实现 Refresh Token 机制） | 无需（需携带有效 Refresh Token） |
| `/api/auth/change-password` | POST | 修改当前用户密码 | 必须 |
| `/api/users` | GET | 获取用户列表（管理员） | 管理员 |
| `/api/users` | POST | 创建新用户（管理员） | 管理员 |
| `/api/users/{id}` | GET/PUT/DELETE | 获取/更新/删除用户 | 管理员（普通用户可更新自己的部分字段） |
| `/api/books` | GET | 查询图书列表（支持搜索、分页、排序） | 必须 |
| `/api/books` | POST | 录入新图书（管理员） | 管理员 |
| `/api/books/{id}` | GET | 获取图书详情 | 必须 |
| `/api/books/{id}` | PUT | 更新图书信息（管理员） | 管理员 |
| `/api/books/{id}` | DELETE | 删除图书（逻辑删除，管理员） | 管理员 |
| `/api/borrow` | POST | 创建借阅记录 | 必须 |
| `/api/borrow/{id}/return` | POST | 归还图书 | 必须（仅管理员或借阅人本人） |
| `/api/borrow` | GET | 查询借阅记录列表（本人或管理员全部） | 必须 |
| `/api/stats` | GET | 获取统计看板数据 | 必须 |
| `/api/logs` | GET | 查询操作日志（管理员） | 管理员 |
| `/api/scanner/scan` | POST | 上传扫描图片/发送 ISBN 字符串触发 AI 录入流程 | 管理员 |

## 4. 系统特性

### 4.1 用户管理

#### 4.1.1 功能需求详情

| 需求编号 | 需求描述 | 优先级 |
|----------|----------|--------|
| FR-001 | 系统必须支持管理员创建、编辑、删除家庭成员账号 | P0 |
| FR-001a | 管理员创建账号时必须设置初始密码，系统应强制该用户在首次登录时修改密码 | P0 |
| FR-001b | 系统应当支持管理员为忘记密码的成员重置密码，重置后的密码同样为临时密码，需强制修改 | P1 |
| FR-002 | 系统必须支持用户登录（用户名/密码） | P0 |
| FR-003 | 系统必须支持角色划分：管理员、普通成员 | P0 |
| FR-004 | 系统应当支持管理员修改其他成员的角色和权限 | P1 |
| FR-005 | 系统应当支持普通成员修改自己的昵称、头像、密码 | P1 |

#### 4.1.2 用例描述

**UC-001：用户登录**

| 项目 | 内容 |
|------|------|
| **用例名称** | 用户登录 |
| **参与者** | 管理员、普通成员 |
| **前置条件** | 用户已注册账号 |
| **后置条件** | 用户获得有效会话（JWT Token），跳转至首页；若密码为初始密码则跳转至首次登录改密页 |
| **触发条件** | 用户访问系统登录页面 |

**基本流：**
1. 用户打开系统登录页面。
2. 系统显示用户名和密码输入框。
3. 用户输入用户名和密码。
4. 用户点击“登录”按钮。
5. 系统验证用户名和密码是否正确。
6. 系统检查用户是否为首次登录（密码标记为临时）。
   - 若是，则生成临时 Token，前端跳转至修改密码页面。
   - 若否，则生成正式 JWT Token 并返回给前端。
7. 前端存储 Token，跳转至系统首页。
8. 用例结束。

**备选流：**
- **5a. 用户名或密码错误**：
  1. 系统提示“用户名或密码错误”。
  2. 记录失败次数（基于用户名或 IP，存储在 Redis 或数据库）。
  3. 若连续失败 5 次，系统锁定该账号 15 分钟，提示“账号已锁定，请 15 分钟后重试”。
  4. 返回步骤 3。
- **5b. 账号已被锁定**：
  1. 系统提示“账号已被锁定，请 15 分钟后重试”。
  2. 用例结束。
- **6a. 首次登录改密**：
  1. 系统弹出修改密码界面，要求输入新密码（强度校验：至少8位，含字母和数字）。
  2. 用户提交新密码后，系统更新密码并清除临时标记，返回正式 Token。
  3. 跳转至首页。

#### 4.2.2 用例描述

**UC-002：通过 ISBN 扫描录入图书**

| 项目 | 内容 |
|------|------|
| **用例名称** | 通过 ISBN 扫描录入图书 |
| **参与者** | 管理员 |
| **前置条件** | 管理员已登录系统 |
| **后置条件** | 图书信息成功入库，系统显示录入成功 |
| **触发条件** | 管理员点击“扫描录入”按钮 |

**基本流：**
1. 管理员在图书管理页面点击“扫描录入”按钮。
2. 系统请求摄像头权限，打开手机摄像头扫码界面。
3. 管理员将摄像头对准图书背面 ISBN 条形码。
4. 系统识别条形码，提取 ISBN 号码。
5. 系统将 ISBN 发送至后端。
6. 后端调用 AI 智能体服务（通过异步任务），AI 智能体依次通过后端代理查询国图 API、豆瓣 API 获取图书元数据。
7. AI 智能体将获取到的元数据（书名、作者、出版社、出版日期、封面 URL、分类、简介等）结构化后返回给后端。
8. 后端将元数据展示在前端预览页面，供管理员确认。
9. 管理员确认信息无误，选择图书品类（默认“单本”），填写物理存放位置（可选）。
10. 管理员点击“确认入库”按钮。
11. 系统将图书信息写入 MySQL 数据库，记录操作日志。
12. 系统提示“录入成功”，并显示新录入的图书详情。
13. 用例结束。

**备选流：**
- **4a. 识别失败**：
  1. 系统提示“识别失败，请重新扫描或手动输入 ISBN”。
  2. 管理员可选择重新扫描或切换至手动输入。
  3. 若手动输入，跳转至 UC-003 备选流。
- **6a. 国图/豆瓣均无数据**：
  1. 系统根据 ISBN 解析出有限信息（如出版社前缀），并在表单中预填。
  2. 系统提示“未找到在线数据，请手动完善”。
  3. 跳转至手动录入模式，仅 ISBN 字段锁定。
  4. 管理员手动填写书名、作者等必填字段。
  5. 跳转至步骤 9。
- **6b. 国图/豆瓣返回数据不一致**：
  1. AI 智能体综合判断，返回最可能的数据，并标注“数据来源：国图/豆瓣”。
  2. 系统展示数据，管理员可手动修改。
  3. 跳转至步骤 9。
- **9a. 管理员发现信息有误**：
  1. 管理员直接在预览页面修改字段。
  2. 点击“确认入库”继续。
- **9b. 图书为套装书**：
  1. 管理员选择品类为“套装”，并填写总册数。
  2. 系统展开子表单，管理员逐册填写分册名、分册 ISBN（可选）。
  3. 跳转至步骤 10。

**UC-003：手动录入图书**
