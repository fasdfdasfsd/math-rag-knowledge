## 1. 引言

### 1.1 目的

本文档旨在明确企业低代码平台中页面设计器及相关功能模块的详细需求，为系统设计、开发、测试及验收提供统一依据。平台以百度 AMIS 框架为参考蓝本，实现可视化拖拽式页面构建、丰富组件生态、API 集成、实时预览及移动端适配等核心能力，降低企业应用开发门槛，提升交付效率。

### 1.2 文档约定

- **需求编号规则**：需求编号采用 `{域缩写}-{序号}` 格式。域缩写定义如下：
  - `F-ED`：页面设计器编辑环境
  - `F-PT`：页面类型管理
  - `F-CL`：组件库管理
  - `F-FC`：表单组件生态
  - `F-FM`：表单核心功能
  - `F-API`：API 接口配置与调用
  - `F-PV`：预览功能
  - `F-MA`：移动端适配
  - `F-VA`：可视化辅助功能
  - `F-EI`：扩展与集成
  - `N-PF`：性能需求
  - `N-SC`：安全需求
  - `N-UB`：可用性需求
  - `N-CP`：兼容性需求
  - `N-MT`：可维护性需求
  - `N-EX`：可扩展性需求
- **优先级定义**：P0（关键需求，MVP 必须交付）、P1（重要需求，第二阶段交付）、P2（增强需求，后续迭代交付）。
- **关键词约定**：本文档中“必须（shall）”表示强制要求，“应该（should）”表示推荐要求，“可以（may）”表示可选要求。
- **用例编号**：用例编号采用 `UC-{序号}` 格式。

### 1.4 产品范围

本需求覆盖以下功能域：
- 页面设计器编辑环境（拖拽布局、实时预览、JSON 配置生成与导出）
- 表单组件生态（60+ 开箱即用组件，涵盖输入、选择、日期、布局、交互、展示等类别）
- 表单核心功能（20+ 控件类型、多模式布局、验证规则、动态字段、条件联动、API 数据源绑定）
- 可视化辅助功能（配置面板联动、模板市场、组件库管理、插件机制、无限级嵌套）
- 扩展与集成（自定义组件注册、主题覆盖、国际化、性能优化）
- API 接口配置与调用、预览功能、移动端适配

**不包含**：后端业务逻辑引擎、工作流引擎、权限管理模块（仅涉及设计器自身权限）、数据持久化存储方案。

## 2. 综合描述

### 2.1 产品前景

企业低代码平台旨在解决企业应用开发中重复性高、交付周期长、技术门槛高等痛点。通过提供可视化拖拽式页面构建能力，使业务人员能够直接参与应用开发，减少对专业开发人员的依赖。平台以百度 AMIS 框架为技术底座，兼容其 JSON Schema 协议和组件体系，确保生成的页面配置可被标准渲染引擎消费，实现“一次设计，多端运行”。

### 2.2 功能概述

系统核心功能包括以下八大能力域：

| 能力域 | 核心功能 | 优先级 |
|--------|----------|--------|
| 页面设计器编辑环境 | 可视化拖拽编辑、实时预览、JSON 配置生成与导出、撤销/重做 | P0 |
| 页面类型管理 | 标准页面、弹出框、侧拉框、浮动框、模板引用、版本管理 | P0 |
| 组件库管理 | 60+ 内置组件、分类展示、搜索过滤、收藏、分组管理 | P0 |
| 表单组件生态 | 8 大类 59 个组件（输入、选择、日期、高级输入、布局、交互、展示、复合） | P0 |
| 表单核心功能 | 20+ 控件类型、三种布局模式、验证规则、条件联动、API 数据源绑定 | P0 |
| API 接口配置与调用 | API 地址配置、HTTP 方法、请求头/参数、数据映射、触发时机、Mock | P0 |
| 预览功能 | 实时预览、独立预览、设备模拟、交互模拟 | P0 |
| 移动端适配 | 响应式布局、设备属性配置、触屏优化、移动端预览 | P0 |
| 可视化辅助功能 | 配置面板联动、模板市场、组件库管理、插件机制、大纲视图 | P1 |
| 扩展与集成 | 自定义组件注册、主题覆盖、国际化、性能优化 | P1 |

### 3.3 软件接口

#### 3.3.1 后端服务 API

所有 API 遵循 RESTful 规范，采用 JSON 格式，需携带 `Authorization: Bearer <JWT_TOKEN>` Header 进行认证。未认证或令牌过期的请求必须返回 401 状态码。所有响应必须符合 3.3.3 节定义的统一信封格式。

**页面管理 API**

| POST | `/api/pages` | 创建页面 | `{ "title": "string", "type": "standard|modal|drawer|popover", "schema": {} }` | 201 | 400（参数无效）, 409（标题冲突） |

#### 3.3.2 前端内部接口

**插件扩展点接口**

```typescript
interface EditorPlugin {
  name: string;
  version: string;
```

```typescript
interface ComponentDefinition {
```

#### 3.3.3 统一 API 响应格式

所有后端 API 必须返回以下标准的 JSON 信封，确保前端能统一处理成功和错误情况。

```json
{
  "success": true,
  "data": { ... },       // 成功的业务数据
  "error": null
}
```

失败响应（至少包含错误码和用户可读消息）：

```json
{
  "success": false,
  "data": null,
  "error": {
    "code": "RESOURCE_NOT_FOUND",
    "message": "请求的页面不存在",
    "details": []       // 可选，用于参数校验失败等场景，列出具体字段错误
  }
}
```

HTTP 状态码必须与响应语义一致（如 200 对应大部分成功，201 对应创建，400 对应客户端错误，401/403 对应认证/授权，404 对应资源不存在，409 对应并发冲突，500 对应服务端错误）。

### 3.4 通信接口

| 需求编号 | 需求描述 | 优先级 |
|----------|----------|--------|
| CI-01 | 前端与后端之间通过 HTTPS 协议通信 | P0 |
| CI-02 | API 请求需携带 JWT Token 进行身份认证，具体格式为 `Authorization: Bearer <token>`。令牌过期时前端必须能自动处理重定向至登录页。 | P0 |
| CI-03 | 设计器预览时通过 API 代理网关转发请求，解决跨域问题。网关必须对转发请求的 URL、方法和参数进行白名单校验，防止 SSRF 攻击。 | P0 |

## 4. 系统特性

### 4.1 页面设计器编辑环境

#### 4.1.1 功能需求

| 编号 | 需求描述 | 优先级 |
|------|----------|--------|
| F-ED-01 | 系统应提供可视化拖拽式编辑界面，用户可从左侧组件面板拖拽组件至中央画布指定位置。当拖拽违反嵌套规则时（如将非容器组件拖入不允许的子区域），必须抛出清晰的视觉禁止信号，阻止放置，并在面板显示原因。 | P0 |
